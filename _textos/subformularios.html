Visão geral

Um controle Subformulários insere um formulário dinâmico que já foi criado no projeto em um formulário dinâmico mestre, adicionando-o como outra linha do projeto e exibindo seu projeto completo ao executar um caso. Para adicionar esse controle ao formulário dinâmico, arraste o ícone a seguir do painel esquerdo e solte-o na Ferramenta de Desenho.

Uma caixa de mensagem Selecione Controle será aberta com uma lista de todos os Formulários Dinâmicos existentes no projeto. Selecione o Formulário Dinâmico que será adicionado ao projeto na lista.

O Subformulário Dinâmico será incluído no projeto com todas as suas linhas exibidas.

O controle de subformulário não permite que o formulário inserido no projeto seja editado. O formulário dinâmico original deve ser alterado para fazer alterações em todos os subformulários que usam o formulário dinâmico.

Quando o formulário dinâmico é renderizado (na visualização ou durante a execução de casos), os subformulários são exibidos como conteúdo de formulário normal.

Ao trabalhar com subformulários, leve em consideração os seguintes pontos:

  • O mesmo formulário só pode ser usado uma vez no mesmo formulário dinâmico (para evitar a duplicação do uso de variáveis e IDs).
  • O subformulário não pode ser editado no formulário dinâmico mestre. Ele é exibido como somente leitura. Para modificar o subformulário, abra-o na Ferramenta de Desenho do Formulário Dinâmico. As alterações feitas no formulário serão refletidas automaticamente em todos os campos do subformulário do dinâmico formulário mestre.
  • Apenas um nível de aninhamento é permitido. Um subformulário não pode conter subformulários.

    Um formulário pode conter vários subformulários (que devem ser diferentes).

  • Para evitar a duplicação de variáveis e IDs com subformulários, uma validação deve ser executada para alertar o usuário sobre quaisquer variáveis/IDs duplicados existentes. A Ferramenta de Desenho do Formulário Dinâmico deve decidir se o projeto pode continuar com as variáveis/IDs duplicados.

Aviso: Tome cuidado ao atualizar o Gestor Express. Devido às alterações feitas na funcionalidade dos subformulários, não é mais possível personalizar os subformulários quando eles são adicionados ao formulário dinâmico mestre (suas propriedades estarão no modo de exibição, portanto nenhuma alteração poderá ser feita). Se você personalizou os subformulários em versões anteriores do Gestor Express (como adicionar JavaScript personalizado, modificar os controles etc.), essas alterações serão perdidas.

Observação: É possível deletar linhas e controles do subformulário, mas lembre-se que se o formulário original for modificado depois que as linhas e controles foram deletados do subformulário no formulário dinâmico mestre, o subformulário será renderizado no Formulário Dinâmico mestre apenas aplicando as alterações feitas no formulário original, portanto as alterações no subformulário não terão efeito.

Propriedades

Clique em qualquer espaço cinza claro no subformulário para visualizar e trabalhar com suas propriedades.

PropriedadeDescrição
Tipoform (somente leitura)
IDO mesmo ID (identificador HTML exclusivo) do formulário pai (somente leitura).
Nome[Somente leitura]
O mesmo nome do formulário pai.
Descrição[Somente leitura]
A mesma descrição definida quando o formulário pai foi criado. Esta propriedade descreve o que é o Formulário Dinâmico.
Modo[Somente leitura]
Modo de exibição do formulário mestre:
  • editar: o subformulário é habilitado para edição pelo usuário final quando o formulário dinâmico mestre é renderizado.
  • visualizar: o controle só pode ser visualizado quando for renderizado. O conteúdo não pode ser editado.
  • desativado: o controle é exibido em cinza para indicar que está desativado. O conteúdo também não pode ser editado.
JavaScript[Somente leitura]
Em subformulários, o código JavaScript é concatenado ao formulário pai, portanto, essa propriedade é somente leitura.
Bibliotecas Externas[somente leitura]
Área onde são mostradas as bibliotecas externas do formulário pai.

Observação: Leve em consideração que se o subformulário possuir código JavaScript em sua configuração, ele será executado toda vez que o formulário dinâmico for carregado.

Propriedades do subformulário

Subformulários são basicamente formulários dinâmicos embutidos em outro formulário. As propriedades de um subformulário são herdadas do formulário dinâmico original e não podem ser modificadas. No entanto, os controles de subformulário têm uma propriedade adicional explicada na próxima seção.

Loja de grade

Disponível em: Gestor Express 3.0.1.5 a 3.0.1.7. Esta propriedade não está mais disponível a partir da versão 3.0.1.8 em diante.

Esta propriedade permite que um controle de subformulário seja relacionado a uma variável de grid, então muitas instâncias do subformulário podem ser adicionadas ao formulário - uma linha após a outra - como linhas de uma grade.

Nota: No momento, esta propriedade relaciona apenas uma variável de grade a um controle de subformulário. Esta opção foi adicionada para fins experimentais na versão 3.0.1.5, mas foi removida a partir da versão 3.0.1.8 em diante.

Para relacionar a variável de grade ao controle do subformulário, selecione o subformulário no projetoer.

Verifique a propriedade de armazenamento de grade e as propriedades de tipo de dados variável e variável serão exibidas abaixo.

Selecione ou crie uma nova variável de grid para associar ao subformulário.
