Links

Os links são usados para exibir um hiperlink em formulários dinâmicos. Defina a URL (endereço web) para onde o link irá redirecionar na propriedade Href. Se necessário, adicione uma dica de onde o link redireciona na dica de propriedade. Por fim, adicione um valor ao link na propriedade valor.

O ícone deste elemento dentro da Ferramenta de Desenho do Formulário Dinâmico é o seguinte

Para adicionar um link ao design do formulário dinâmico, arraste e solte-o em qualquer uma das linhas existentes.

Defina a URL para a qual o link será redirecionado dentro de suas propriedades.

A propriedade Href

Esta propriedade define o endereço de um link. Quando o link for clicado, o navegador da web abrirá uma nova guia ou janela (dependendo da configuração do navegador da web) que direciona para esse endereço.

Se um endereço não tiver um protocolo, como https:// ou ftp://, http:// será automaticamente anexado ao endereço, portanto, um endereço como www.gestorexpress.com.br se tornará http://www.gestorexpress.com.br.

Ao renderizar o Formulário Dinâmico na visualização ou executar um caso, o texto na cor azul será o link para a URL especificada nesta propriedade. Os usuários devem clicar neles para abrir a página da web.

Qualquer tipo de URL pode ser usado, como:

  • Exemplo http: http://www.example.com.br/
  • Exemplo https: https://pt.wikipedia.org
  • Exemplo mailto: mailto:silvia.alvarado@gestorexpress.com.br
  • Exemplo ftp: ftp://192.168.1.20

Além disso, endereços IP podem ser adicionados nesta propriedade, como 127.0.0.1, http://192.168.1.77, etc.

Também é possível redirecionar páginas dinamicamente usando variáveis de caso no código JavaScript. Por exemplo:

  • Crie a variável casehrefRedirect
  • Crie um novo gatilho e defina a url para onde a página será redirecionada usando o código
    • @@hrefRedirect = "http://www.gestorexpress.com.br";
  • Atribua o gatilho antes do Formulário Dinâmico nas etapas da tarefa.
  • Adicione o nome da variável na propriedade "Href" do Formulário Dinâmico: @@hrefRedirect.

    • Ou adicione o seguinte código JavaScript para abrir o URL em uma nova página:

    javascript: window.open('@#href', '_blank');

    Observação: Leve em consideração as Regras de Digitação das variáveis de caso para usá-las no código.

  • Execute um caso e clique no link do Formulário Dinâmico. Outra guia será aberta e redirecionará para a página inicial do Gestor Express.

Exemplo do controle link

Para este exemplo, arraste e solte um controle "Link" na tela do Formulário Dinâmico. O resultado da adição do controle será o seguinte.

Clique no espaço vazio do controle para exibir suas propriedades no painel do lado esquerdo na guia "Controles da Web".

A primeira propriedade é a propriedade "id" que é o identificador exclusivo do controle. Para este exemplo, a propriedade "id" será usada com código JavaScript para adicionar funcionalidade e comportamento ao controle.

Para isso, clique na borda cinza claro do Formulário Dinâmico (que é o espaço fora dos controles) para exibir as propriedades do Formulário Dinâmico no painel esquerdo. Vá para a propriedade "javascript" que possui um botão "editar" que abrirá um editor de JavaScript.

No editor, adicione o seguinte código que definirá o texto do link:

jQuery("#link0000000001").setValue('http://www.gestorexpress.com.br');

Observe que o "id" está sendo usado na consulta para definir o valor no controle. A propriedade "id" pode ter um nome diferente em vez de "link0000000001".

Se renderizado neste ponto, o controle não mostrará nada, portanto, a propriedade "exibir texto" deve ter um nome apropriado para o link. Por exemplo, o texto adicionado na propriedade "exibir texto" pode ser: "Clique aqui". Quando renderizado, o controle terá o texto adicionado e levará ao link colocado no código JavaScript.

A propriedade "nome" é uma referência de nome de campo HTML para o controle.

Executando JavaScript em Links

Até o Gestor Express 3.2.2, a propriedade "href" dos links pode ser definida com o método setHref(). No entanto, esse método adiciona automaticamente "http://"se não tiver um protocolo, o que é um problema ao tentar executar o código JavaScript em um link. Para contornar esse problema, use jQuery padrão para definir a propriedade "href":

$("# link-id ").find("a").prop("href", "javascript: código ")

Por exemplo, um formulário dinâmico tem um link com o ID "clientLink". Quando esse link é clicado, ele deve executar uma função JavaScript personalizada chamada redirectPage(), que possui código para primeiro chamar saveForm() para salvar o conteúdo atual do Formulário Dinâmico e, em seguida, definir o local para redirecionar para um Formulário Dinâmico na mesma tarefa cujo UID é "95038966757f286de4cc9f6008453249 " e cuja posição de passo é 2.

Para redirecionar para outra etapa em uma tarefa, o redirecionamento deve ocorrer dentro do mesmo quadro que o Formulário Dinâmico atual, portanto, a propriedade de destino do link é alterada de "_blank"(que abre uma nova janela ou guia) para "".

$ ( "#clientLink" ). encontre ( "um" ). prop ( "href", "javascript: redirectPage()" ) ;

$ ( "#clientLink" ). encontre ( "um" ). prop ( "destino", "" ) ;

function redirectPage ( ) {

$ ( "form" ). salvarFormulário ( ) ;

location = "../cases/cases_Step?TYPE=FORMULÁRIO DINÂMICO&UID=95038966757f286de4cc9f6008453249&POSITION=2&ACTION=EDIT" ;

}

Configurando Links Relativos

Até o Gestor Express 3.2.2, por padrão, os controles Link adicionam "http://" ao usar o método setHref(). Se precisar usar um link relativo, como../login/loginaquele que não contém o endereço completo, procure a tag "a" dentro de um link dive use o método.prop() do jQuery para definir sua propriedade "href" para o endereço relativo.

Por exemplo, o código JavaScript a seguir redireciona o navegador da web para o endereço relativo..login/loginem uma nova guia ou janela (dependendo da configuração do navegador da web):

$ ( "#clientLink" ). encontre ( "um" ). prop ( "href", "../login/login" ) ;

Manipulando links em gatilhos

O Gestor Express 3 não permite que variáveis sejam associadas a campos de link, como outros campos, portanto, o valor de um link não pode ser definido por meio de sua variável associada. No entanto, as propriedades href e de texto exibido de um campo de link podem ser definidas com variáveis. Na Ferramenta de Desenho do Formulário Dinâmico, clique no campo link e coloque as variáveis nas propriedades.

Neste exemplo, a @@productNamevariável é colocada na propriedade de texto exibido do link e a @@productUrlvariável é colocada na propriedade href do link.

Se as variáveis @@productNamee @@productUrlforem definidas em algum ponto anterior no caso, o campo de link conterá os valores nas variáveis. As variáveis @@productNamee @@productUrlpodem ser definidas por campos em um Formulário Dinâmico anterior ou podem ser definidas com código PHP em um gatilho.

Por exemplo, o seguinte código de gatilho pode ser usado para definir as variáveis @@productNamee:@@productUrl

if ( @@ job == 'sprocket' ) {

@@ productName = '3cm flanged sprocket' ;

@@ productUrl = 'http://www.toolworld.com/products/3cm_flanged_sprocket.html' ;

}

else {

@@ productName = 'porca de 20 mm' ;

@@ productUrl = 'http://www.toolworld.com/products/20mm_lug_nut.html' ;

}

O gatilho acima deve ser configurado para ser executado antes do Formulário Dinâmico ou em algum ponto anterior.
