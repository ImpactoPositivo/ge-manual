Visão geral

Campos ocultos são usados para armazenar dados - não vistos pelo usuário - no formulário dinâmico. Como os campos do Formulário Dinâmico normais, os campos ocultos podem ser usados para passar dados de ou para o Formulário Dinâmico, que não precisam ser vistos pelos usuários. Este controle é representado na barra de ferramentas da Ferramenta de Desenho do Formulário Dinâmico pelo ícone a seguir:

Esse tipo de controle só é visível dentro da Ferramenta de Desenho do Formulário Dinâmico e não ao executar um caso ou na visualização do Formulário Dinâmico. Ele não possui caixa de entrada, mas a variável atribuída a ele pode ser usada como curinga para operações com outros elementos do formulário dinâmico.

Gerenciando controles ocultos

Após adicionar um controle oculto ao modelo de formulário, é possível definir sua funcionalidade e comportamento no painel de propriedades.

As propriedades definidas por padrão são as seguintes:

  • tipo Definido por padrão como "hidden". Esta propriedade não pode ser modificada.
  • id Definido por padrão como hiddenVar00X onde "X" representa a numeração correspondente do controle no projeto. Esta numeração começa em 1. Após uma variável ser relacionada a este controle, o ID muda para o nome da variável, porém é possível definir um novo ID para trabalhar com o controle.
  • Conexão BD Definido por padrão como "PM Database". Leve em consideração que se nenhuma consulta SQL for definida na propriedade "sql", nenhuma execução será feita. Além disso, quando uma variável está relacionada a este controle que possui sua própria definição da conexão com um banco de dados (um banco de dados externo ou banco de dados do Gestor Express) esta propriedade herda a definição da variável relacionada no controle.

Propriedades ocultas

As propriedades para gerenciar esse controle são as seguintes:

PropriedadeDescrição
Modelohidden (somente leitura)
VariávelClique na opção ... para selecionar a variável da lista de variáveis disponíveis. Consulte esta seção para saber mais sobre como relacionar uma variável a um controle.
Somente variáveis dos seguintes tipos de dados podem ser selecionadas:
  • float (com opções)
  • inteiro
  • string
  • Booleano // string PHP que pode ser avaliada como verdadeira ou falsa
  • Data e hora //no formato PHP, mas o controle deve ser capaz de analisá-lo para data e hora
Tipo de dados da variávelApós uma variável ter sido relacionada ao controle, esta propriedade mostra o tipo de dado da variável (somente leitura).
ID[Requeridos]
Identificador exclusivo de campo e HTML.
Valor padrãoValor definido por padrão quando o valor recuperado é nulo.
Para saber mais sobre valores padrão, valores nulos e algumas outras considerações sobre essa propriedade, consulte esta documentação .
Conexão BDPara um controle oculto, caso precise armazenar um valor recuperado de um banco de dados, selecione no menu suspenso desta propriedade a conexão do banco de dados que será utilizada (a conexão já deve estar criada no processo, consulte esta seção para saber mais sobre isso).
SQLInsira nesta propriedade, a consulta SQL para preencher a opção do controle do banco de dados. Para um controle oculto, o valor armazenado no campo será o primeiro valor recuperado do banco de dados.
OpçõesDefina nesta propriedade as opções que serão utilizadas para definir o valor utilizado no campo. Para um controle oculto, o valor armazenado no campo será o primeiro valor definido nas opções.
Leia esta seção para saber como esse controle gerencia os valores quando há opções definidas para o controle ou no controle.

Este exemplo mostrará como um controle "Oculto" pode ser usado de muitas maneiras diferentes. Portanto, foi criado um processo simples no qual existem duas tarefas e um evento de notificação. Recrie o processo de exemplo usando a imagem abaixo como guia.

Crie um Formulário Dinâmico com dois controles "texbox" como visto na imagem abaixo.

Adicione um controle "oculto" arrastando-o e soltando-o do Controle da Web no painel esquerdo.

Imediatamente após a adição do controle, a nova janela Criar variável aparecerá. Crie uma variável para este controle como visto na imagem abaixo.

O Formulário Dinâmico final deve se parecer com a imagem mostrada abaixo.

Agora clique em qualquer espaço vazio do controle "oculto" para exibir suas propriedades no painel esquerdo.

A primeira propriedade é a propriedade "id" que pode ser usada com código Javascript para adicionar funcionalidade e comportamento ao controle. Para isso, clique na borda cinza claro do Formulário Dinâmico (que é o espaço fora dos controles) para exibir as propriedades do Formulário Dinâmico. Quando as propriedades forem exibidas localize a propriedade "javascript" e clique no botão "editar" para abrir um editor de Javascript.

Uma vez no editor, adicione o seguinte código Javascript:

jQuery("#hiddenVar").setValue("TestValue: GestorExpress")

Clique no botão "Gravar" para armazenar o código adicionado recentemente.

Antes de testar o controle, configure o evento intermediário no processo clicando com o botão direito sobre ele e selecionando suas propriedades.

Configure o evento de email com as especificações mostradas na imagem abaixo. Observe que a variável de controle "oculta" está sendo adicionada no conteúdo do email.

Para testar o controle vá em Início => Novo Caso e então dê um duplo clique no processo que possui o Formulário Dinâmico criado acima.

Como visto na imagem abaixo, o controle "oculto" não é mostrado ao executar um caso, mas está funcionando em segundo plano. Após preencher as informações solicitadas clique no botão "Enviar".

Uma vez clicado aparecerá a janela da próxima tarefa, clique no botão "Continuar" e a notificação por e-mail enviará o e-mail criado.

Verifique o e-mail que foi enviado. Pode-se observar que o valor de controle "oculto" adicionado no código Javascript pode ser encontrado no conteúdo do e-mail conforme imagem abaixo.

Para este exemplo a propriedade "valor padrão" mostra um valor definido pelo usuário como: "Confirme seu endereço de e-mail clicando neste link: www.gestorexpress.com". Execute este caso para receber uma notificação por email com o valor padrão do controle.

Observação: O valor padrão não terá efeito no controle se o código Javascript adicionado antes ainda estiver funcionando, portanto, é recomendável apagar ou comentar o código antes de executar o processo.

JavaScript em controles ocultos

Para saber como gerenciar campos Formulário Dinâmico usando JavaScript, consulte JavaScript em Formulário Dinâmicos.

Os controles ocultos no Gestor Express têm os seguintes componentes de campo:

  • Texto: O texto dentro do campo.
  • Valor: O valor armazenado para o campo. Para controles ocultos, o texto e o valor são os mesmos.

Métodos JavaScript

Alguns dos métodos JavaScript para manipular controles ocultos incluem:

MétodoDescrição
jQuery("#fieldID").getControl()Use esta função para obter o campo de entrada do controle, em vez de seu DIV que é obtido com$("#id")
jQuery("#fieldID").getValue()Retorna o valor do campo oculto. Observe que getValue() e getText() retornam a mesma coisa.
jQuery("#fieldID").getText()Retorna o texto do campo oculto. Observe que getValue() e getText() retornam a mesma coisa.
jQuery("#fieldID").setValue("novo valor")Define um novo valor no campo. Observe que, para controles ocultos, setText() é o mesmo que setValue().
jQuery("#fieldID").setText("novoTexto")define um novo texto no campo. Observe que, para controles ocultos, setText() é o mesmo que setValue().

Para ver exemplos de código para esses métodos, consulte Funções e métodos JavaScript.
