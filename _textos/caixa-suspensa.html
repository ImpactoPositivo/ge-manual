Visão geral

Um controle de caixa suspensa permite que os usuários selecionem uma única opção de uma lista de opções predeterminadas com menu dropdown.

Para adicionar este controle no Formulário Dinâmico, vá para a caixa de ferramentas de controle no lado esquerdo da Ferramenta de Desenho do Formulário Dinâmico, arraste e solte o seguinte ícone:

Ao adicionar este controle ao modelo, é possível relacioná-lo a uma variável com opções criadas ou uma variável que recupera dados de uma consulta SQL, conforme explicado nesta seção. Os dados da variável serão mostrados ao clicar na seta para baixo do controle. Se as opções do menu suspenso forem recuperadas de uma consulta SQL, as opções não serão mostradas dentro da Ferramenta de Desenho. As opções da Caixa Suspensa são mostradas apenas quando o Formulário Dinâmico é renderizado dentro do "Visualizar" e ao executar um caso. Ao executar um caso:

As caixas suspensas são usadas porque economizam espaço, mas não permitem seleções múltiplas e o usuário é obrigado a usar o mouse para exibir a lista de opções disponíveis.

Uma opção pode ser selecionada clicando uma vez na caixa suspensa para exibir a lista de opções e, em seguida, clicando uma segunda vez na opção a ser selecionada. Alternativamente, uma opção pode ser selecionada mantendo pressionado o botão direito do mouse para exibir a lista e, em seguida, movendo o cursor do mouse para uma opção e soltando o mouse.

Para selecionar uma opção com o teclado, use TAB ou SHIFT+TAB para trazer o foco para a caixa suspensa. Pressione a tecla de seta PARA CIMA ou ESQUERDA para selecionar a opção anterior na lista. Pressione a tecla de seta PARA BAIXO ou DIREITA para selecionar a próxima opção na lista. Pressione HOME para selecionar a primeira opção da lista e END para selecionar a última opção. Pressionar uma tecla de letra ou número moverá a seleção para a primeira opção cujo rótulo comece com essa letra ou número. Pressionar a mesma letra ou número novamente move para as opções subsequentes que começam com essa letra ou número. Não há como exibir a lista de opções com o teclado, mas uma vez que a lista é exibida com um único clique, o teclado pode ser usado para mover a seleção. Para selecionar a opção atual e ocultar a lista, pressione ENTER ou TAB.

Gerenciando caixas suspensas

Depois de adicionar o controle a um modelo de formulário, é obrigatório definir suas propriedades para dar-lhe funcionalidade em relação ao seu comportamento, aparência e os dados mostrados em suas opções. Para fazer isso, clique no controle e vá para o painel esquerdo onde são mostradas suas propriedades:

As propriedades definidas por padrão são as seguintes:

  • tipo Definido por padrão como "dropdown". Esta definição não pode ser modificada.
  • id Definido por padrão como dropdownVar00X, onde "X" representa a numeração correspondente do controle no projeto. Esta numeração começa em 1. Após uma variável ser relacionada a este controle, o ID muda para o nome da variável, porém é possível definir um novo ID para trabalhar com o controle.
  • rótulo Definido por padrão como "dropdown_X" onde "X" representa a numeração correspondente do controle no desenho. Esta numeração começa em 1.
  • modo de exibição Definido por padrão como "mestre". Consulte esta documentação para obter mais informações sobre os modos disponíveis para este controle.
  • fonte de dados Definido por padrão como "banco de dados". Consulte esta documentação para obter mais informações sobre a fonte de dados disponível para esse controle.
  • Conexão de banco de dados Definido por padrão como "PM Database". Leve em consideração que se nenhuma consulta SQL for definida na propriedade "sql", nenhuma execução será feita. Além disso, quando uma variável está relacionada a este controle que possui sua própria definição da conexão com um banco de dados (um banco de dados externo ou banco de dados do Gestor Express) esta propriedade herda a definição da variável relacionada no controle.

Exemplo de controle de caixa suspensa

Para este exemplo, adicione um controle "Caixa suspensa" arrastando e soltando-o no modelo do Formulário Dinâmico.

Após adicionar o controle, a janela Criar variável aparecerá imediatamente. Crie uma nova variável ou selecione uma variável já criada. Para este exemplo, crie uma variável chamada "país", digite string. Termine de criar a variável clicando no botão "Gravar".

Agora, exiba as propriedades do controle, clicando em qualquer espaço vazio do controle e suas propriedades serão exibidas no painel esquerdo.

Para este caso altere a propriedade "rótulo" para "país:".

A próxima propriedade é "valor padrão" não é aconselhável adicionar um valor por padrão em uma lista suspensa porque perderá a função do controle. Como no exemplo, adicione o texto "ESCOLHA UM" que será o valor padrão e mostrará apenas este texto.

A propriedade "informativo" mostrará o texto de ajuda que é exibido quando nenhuma opção é selecionada. Por exemplo, adicione "Selecione seu país". e isso será mostrado dentro do controle para orientar o usuário. Depois que uma opção for selecionada, o espaço reservado desaparecerá.

Outra maneira de mostrar ajuda é adicionar texto na propriedade "dica". A dica será mostrada em um ícone colocado no canto direito do controle.

Clique na propriedade "obrigatório" para adicionar um asterisco no rótulo para indicar que o campo é obrigatório.

A próxima propriedade é "modo de exibição" que possui diferentes opções para dar funcionalidade ao controle. Observe a imagem abaixo:

Configurando Campos Dependentes

Observação: Para definir a dependência, basta trabalhar com as consultas das variáveis atribuídas aos controles e as variáveis que definem a dependência (veja o exemplo desta seção).

As listas suspensas podem ser definidas como campos dependentes de outros campos com base em seus estados e valores.

Para estabelecer essa dependência, as variáveis normalmente devem ser definidas com uma instrução SQL SELECT para preencher os campos com valores de um banco de dados, que será mostrado assim que o formulário dinâmico estiver renderizando na visualização ou executando o formulário.

Por exemplo, para a dependência país-estado-localização, vamos adicionar três listas suspensas na Ferramenta de Desenho do Formulário Dinâmico e relacioná-las às variáveis PAISES, ESTADOS e LOCALIZAÇÃO

A variável PAISES e tem a seguinte sentença SQL:

SELECT ic_uid, ic_name FROM ISO_COUNTRY

Observação: O identificador * não é suportado em consultas SQL.

A variável ESTADO tem a sentença SQL:

SELECT is_uid, is_name FROM ISO_SUBDIVISION

WHERE ic_uid = "@#COUNTRIES" ORDER BY is_name

E para a variável LOCALIZACAO:

SELECT il_uid, il_name FROM ISO_LOCATION

WHERE ic_uid = "@#COUNTRIES" AND is_uid = "@#SUBDIVISION"

Observação: Leve em consideração essas regras de digitação para usar variáveis de caso nas consultas.

Para a dependência e a recuperação correta dos dados, a consulta deve diferenciar maiúsculas de minúsculas. Isso significa que se o nome da tabela ou o nome dos campos contiverem letras maiúsculas ou minúsculas no banco de dados, a consulta deverá obrigatoriamente ter as mesmas considerações.

Em seguida, relacione cada menu suspenso adicionado no formulário às variáveis PAISES, ESTADO e LOCALIZACAO. A dependência será definida automaticamente.

Se tiver problemas para definir os campos dependentes, leve em consideração o seguinte:

  • O nome das variáveis pode não ter respeitado o nome dos campos ou da tabela definida no banco de dados (já que a consulta deve diferenciar maiúsculas de minúsculas).
  • A dependência dos controles pode não ter sido definida corretamente.
  • Leve em consideração que "ic_uid="@#COUNTRIES"" não recupera nenhum dado ao executar as consultas. Deve haver um espaço em branco entre o símbolo de igual e as variáveis para que a expressão correta seja "ic_uid = "@#COUNTRIES""

Observação: Os campos dependentes não estão disponíveis quando um controle é definido no modo de visualização.

Procedimentos armazenados do MSSQL e campos dependentes

Os procedimentos armazenados do SQL Server podem ser usados para preencher os controles suspensos. Um procedimento armazenado pode ser chamado da propriedade SQL do controle de caixa suspensa da seguinte maneira:

Nome do procedimento EXEC

Além disso, os campos dependentes podem ser definidos usando procedimentos armazenados. Para fazer isso, o procedimento armazenado do campo dependente deve ser projetado para passar um parâmetro. Por exemplo:

CREATE PROCEDURE nome-do-procedimento ( @Parameter1 nvarchar(255) ) BEGIN

SELECT id, nome FROM escritórios WHERE ID = @Parameter1;

FIM

Uma vez criado, ele pode ser chamado da seguinte maneira:

EXEC procedure-nam @@Parameter1-Value

Onde @@Parameter1-Value é a variável na qual a consulta será baseada para recuperar informações.

Exemplo

Vamos criar uma dependência Category - Subcategory usando as tabelas ProductCategory e ProductSubcategory do banco de dados de exemplo AdventureWorks2012 .

Primeiro, crie um procedimento armazenado simples para o menu suspenso "Categoria".

CRIAR PROCEDIMENTO dbo . productPro1

AS

SELECT top 28 ProductCategoryID , Name

FROM Production . Categoria de Produto;

Este procedimento recuperará o ID e o nome dos primeiros 28 registros na tabela ProductCategory . Depois disso, crie um segundo procedimento armazenado que será usado na lista suspensa "Subcategoria".

CRIAR PROCEDIMENTO dbo . productPro2 ( @Param1 nvarchar ( 255 ) )

SELECT top 28 ProductCategoryID , Name

FROM Production . ProductSubcategory WHERE ProductCategoryID = @Param1;

Observe que o segundo procedimento armazenado passa um parâmetro de comparação para a consulta. Ele passará a categoria selecionada como parâmetro e recuperará o ID e o nome das Subcategorias com base na seleção da categoria.

Depois que os dois procedimentos forem criados, atribua-os aos controles suspensos. Clique no menu suspenso "Categoria" e procure a propriedade "Conexão DB" em suas propriedades, selecione a conexão do banco de dados SQL Server e coloque a seguinte consulta dentro da propriedade SQL .

produto executivoPro1

Depois disso, repita os mesmos passos com o menu suspenso "Subcategoria", mas coloque a seguinte consulta na propriedade SQL em vez da acima:

exec productPro2 @@dropdownVar001

Observe que o procedimento "productPro2" depende de um parâmetro para recuperar valores, este parâmetro deve ser a variável suspensa "Category" (@@dropdownVar001) para definir a dependência.

Verifique a dependência indo ao "Modo de visualização" ou iniciando um caso. As subcategorias exibidas dependerão da opção selecionada na categoria.

Desativando opções em caixas suspensas

As opções podem ser desabilitadas em uma caixa suspensa usando a configuração da propriedade desabilitada das opções para um valor de "disabled". Por exemplo, uma caixa suspensa com o ID "selectCountry" tem uma opção com um valor de "MX"e um rótulo de "Mexico". O código JavaScript a seguir desativa essa opção na caixa suspensa:

$ ( "#selectCountry" ) . find ( "opção[valor=MX]" ) . attr ( "desativado" , "desativado" ) ;

Lembre-se de que desabilitar uma opção não altera a opção selecionada no momento, mas impede que o usuário selecione essa opção no futuro. Use control.setValue() para selecionar uma opção diferente.
