Visão geral

Projetos e processos podem ser importados e exportados de uma instalação do Gestor Express para outra. Esses procedimentos apenas importam ou exportam a definição do processo, que inclui Formulários Dinâmicos, Documentos de Entrada, Documentos de Saída, Gatilhos e contas de grupo. No entanto, eles não importam ou exportam contas de usuário, funções, nenhum caso em andamento, regras de atribuição, supervisores de processo, Tabelas e Tabelas de relatórios (Tabelas GE e As Tabelas de Relatórios possuem interface própria para exportá-las e depois importá-las novamente).

Além disso, também é possível exportar modelos de processos para arquivos "*.bpmn" que posteriormente podem ser abertos em outras ferramentas BPMN. Além disso, dentro do Gestor Express, a opção Importar da modelagem permite importar arquivos "*.bpmn" que contenham modelos de projetos BPMN criados dentro de ferramentas externas.

Exportando um projeto

Aviso: evite incluir o caractere de barra (/) no nome do modelo de processo porque a exportação do processo pode falhar. Este é um problema conhecido que será corrigido nas próximas versões do Gestor Express.

Exporte um processo seguindo as etapas a seguir:

  • Vá para a guia Modelagem e selecione o modelo de processo a ser exportado. Após selecioná-lo, o botão Exportar será habilitado na barra de ferramentas. Clique nele para exportar o modelo de processo.
  • O modelo de processo exportado será baixado automaticamente para o computador do usuário. Normalmente, o arquivo baixado estará localizado na pasta Downloads, a menos que o usuário tenha personalizado o destino dos downloads do navegador.
  • Um modelo de processo também pode ser exportado clicando com o botão direito do mouse sobre o modelo de processo e selecionando a opção Exportar como visto na imagem abaixo.

Exportar um modelo de processo do mapa de processos

Também é possível exportar um modelo de processo do Mapa de Processos seguindo os passos a seguir:

Abra um modelo de processo clicando duas vezes nele para exibi-lo no Mapa do Processo (Um modelo de processo também pode ser aberto clicando na opção Editar).

Uma vez aberto o Mapa de Processos, selecione a opção "Exportar Modelo de Processo" da barra de ferramentas Superior.

O download de um arquivo "*.pmx" será iniciado automaticamente. Salve o arquivo após o término do download. O arquivo "*.pmx" exportado pode então ser importado para outra instalação/espaço de trabalho do Gestor Express. Leve em consideração que todos os arquivos exportados contêm a definição XML do processo que inclui o seguinte:

  • Variáveis
  • Tipos de mensagem
  • Formulário Dinâmicos
  • Documentos de entrada
  • Documento de saída
  • Gatilhos
  • Conexões de banco de dados
  • Modelos
  • Arquivos públicos
  • Permissões
  • Rastreador de Caso
  • Supervisores (eles são preservados desde que sejam Grupos de Supervisores)
  • Grupos atribuídos e sua definição na guia admin. Os membros dos grupos não são exportados. Os usuários devem ser atribuídos assim que o processo for importado

Leve em consideração que os elementos não listados não serão exportados. Por exemplo, os usuários atribuídos não podem ser exportados e, como mencionado anteriormente, as tabelas de relatórios têm sua própria interface para importar/exportar tabelas, de modo que não são incluídas no arquivo "*.pmx". Para exportar um modelo de processo (arquivos.bpmn), revise a seção Importando e exportando um modelo de processo.

Importando um modelo de processo

Primeiro, considere o seguinte:

  • Os arquivos pertencentes ao modelador BPMN do Gestor Express têm a extensão .pmX.
  • Use a mesma opção para importar arquivos "*.pm", "*.pmx" ou "*.bpmn" dentro do Gestor Express.
  • Para importar um modelo de processo processo do Gestor Express siga os seguintes passos:

Observação: Ao importar um processo que não existe em um ambiente, a data de criação do novo processo muda para a data atual.

  • Na guia Modelagem, selecione o botão Importar.
  • Uma nova janela dentro do Modelador será aberta onde o arquivo a ser carregado deve ser selecionado. Clique no ícone do arquivo para abrir o navegador de arquivos.
  • Selecione o arquivo "*.pmx" a ser importado. O arquivo pode ser encontrado na pasta Downloads (a menos que esteja configurado para ser armazenado em uma pasta diferente).Uma vez que o arquivo é selecionado clique no botão Abrir como visto na imagem abaixo.
  • O nome do arquivo será mostrado na janela Processo de Importação, mas ainda não foi importado. Lembre-se que o arquivo que está sendo carregado deve ter extensão .pmx ou .bpmn (somente para modelos), caso contrário será exibida uma mensagem de erro e o arquivo não será carregado. Se a extensão estiver correta, clique no botão Upload para importar o modelo de processo.

  • Uma vez que o arquivo é carregado, ele abre automaticamente o modelo de processo no Mapa do Processo. Por fim, todos os arquivos importados são mostrados na lista de projetos ou no Modelador. Agora, esse processo pode ser trabalhado alterando ou apagando qualquer um de seus elementos.

Os elementos que serão importados junto com o modelo de processos são variáveis, Formulários Dinâmicos, documentos de entrada, documentos de saída, Gatilhos, templates e grupos (sem usuários). As Tabelas utilizadas no modelo de processo, usuários e supervisores de processo não serão importadas. Portanto, é recomendável criar um backup desses elementos ao exportar um modelo de processo. Tenha atenção especial quando o modelo de processo tiver tabelas de relatórios e tabelas Ge. Lembre-se que elas possuem uma interface própria para importação e exportação de tabelas, localizada em ADMIN => Configurações => Tabelas GE. Para saber como exportar e importar tabelas GE e tabelas de relatórios, leia esta seção e a seção a seguir.

Ao importar um processo para outra instalação/espaço de trabalho do Gestor Express, é possível alterar o UID do processo ou manter o UID como está. Então haverá duas opções que são:

Observação: Leve em consideração que esta nova janela que pergunta se deseja criar um novo UID de processo ou manter o UID de processo aparece sempre que um processo é importado. Como os processos devem ter um nome e UID exclusivos, use esta janela para manter melhor os processos.

  • Criar um novo UID para este modelo de processo: Cria um novo UID para que os processos sejam diferenciados. Este novo UID será aleatório e terá 32 caracteres, que todos os UIDs devem ter.
  • Manter UID do modelo de processo importado: Mantém o UID atual do processo que está sendo importado para a instalação/espaço de trabalho.

Observação: Os processos são únicos, são identificados por UID e nome.

Importando modelos de processos com eventos de e-mail

Ao importar modelos de processos que contêm eventos de email, como eventos de email intermediários ou eventos de email finais para uma nova instalação/espaço de trabalho, a referência ao servidor de email original é substituída pelo servidor de email padrão da nova instalação/espaço de trabalho no qual o processo está sendo importado.

Isso significa que não é necessário configurar nenhum evento de e-mail novamente. Caso a nova instalação/espaço de trabalho não tenha um servidor de e-mail configurado, a referência ao servidor de e-mail não será configurada.

Importando um modelo de processo existente dentro do Gestor Express

Ao importar um processo para outra instalação/espaço de trabalho existe a possibilidade de já existir um processo com o mesmo nome em sua definição. Quando isso acontecer, uma nova janela aparecerá onde as duas opções a seguir precisam ser consideradas:

  • Crie um processo completamente novo sem alterar o processo atual.
  • Atualize o processo atual, substituindo todas as tarefas e etapas.

Ambas as opções serão explicadas nas seções adiante.

Observação: Ao importar um processo que já existe em um ambiente, o processo mantém a data de criação do processo do ambiente.

Criar um modelo de processo completamente novo sem alterar o processo atual

Esta é a primeira opção na janela que é mostrada:

Esta opção cria um processo completamente novo com diferentes UIDs para Formulário Dinâmicos, documentos de entrada, documentos de saída, etc. A atribuição de grupo a tarefas também será importada (somente os nomes do grupo; se o nome do grupo não existir no workspace, ele é criado em Admin => Usuários => Grupos). Infelizmente, as informações que serão perdidas e precisarão ser recriadas são as tabelas de relatórios, tabelas GE e atribuições de usuários.

Quando esta opção for selecionada, um novo modelo de processo será criado e aberto no Mapa de Processos. O nome do novo processo tem o formato "Cópia de - Nome_do_processo - Data, Hora"

Atualize o modelo de processo atual, substituindo todas as tarefas e etapas

Esta opção significa sobrescrever completamente o processo existente. Todas as tarefas, Formulário Dinâmicos, documentos de entrada e documentos de saída terão os UID's do processo importado. Essa opção é útil se estiver desenvolvendo uma nova versão de um processo em um servidor de desenvolvimento e, em seguida, precisar substituir o processo atual sendo usado em um servidor de produção. O nome do processo importado terá o mesmo formato do caso anterior.

Algumas precauções precisam ser tomadas ao substituir processos que possuem casos ativos. Se o processo existente contiver tarefas com casos ativos (em andamento), o modelo de processo NÃO poderá ser atualizado com um processo que não contenha essas tarefas e a seguinte mensagem será exibida:

Se for necessário manter um registro histórico de casos anteriores, não há garantia de que os registros estarão acessíveis no processo atualizado.

Importar processos com grupos existentes

Aparecerá uma nova janela que inclui as duas opções a seguir, que precisam ser consideradas:

  • Renomeie os grupos importados
  • Mesclar os grupos importados, com os grupos locais pré-existentes (nenhuma alteração será feita nos grupos locais)

Renomeie os grupos importados

Para esta opção, a seguinte janela será mostrada ao usuário:

Esta opção permite ao usuário renomear os grupos que estão sendo importados. Por exemplo, se houver um grupo chamado "GrupoTeste" então selecionando a primeira opção, como visto na imagem acima, um novo grupo será criado com o nome de "GrupoTeste1".

Mesclar os grupos importados com os grupos locais pré-existentes

Esta opção é mostrada quando o modelo de processo possui um grupo atribuído a alguma das tarefas com o mesmo nome de um dos grupos já criados no Gestor Express.

Se o usuário escolher esta opção, ele mesclará os grupos e manterá o nome do grupo. Por exemplo, se houver um grupo chamado "Funcionários" e o processo que deseja ser importado também tiver um grupo "Funcionários", ele mesclará os grupos. Leve em consideração que os grupos podem ser importados ou mesclados, mas os usuários dentro dos grupos não são importados, portanto, eles devem ser reatribuídos manualmente. Observe que os grupos que estão sendo mesclados serão listados na opção Mesclar, para que o usuário saiba quais grupos estão sendo mesclados.

Importando e exportando um modelo de processo

Um modelo de processo pode ser exportado como um arquivo BPMN e pode ser importado para um aplicativo BPMN externo que suporte esse formato.

Da mesma forma, modelos BPMN criados em diferentes aplicativos BPMN podem ser exportados e importados para o Gestor Express, desde que o arquivo tenha uma extensão .bpmn e atenda aos padrões BPMN. Os modelos exportarão os seguintes elementos:

  • Entradas
  • Piscinas
  • Pistas
  • Tarefas
  • Comentários

Observe que a configuração da tarefa NÃO será incluída.

Aviso: Modelos que incluem objetos de dados (controles de entrada e/ou saída de dados) podem ser corrompidos quando o usuário usa a opção Exportar modelo. Este é um problema conhecido que será corrigido nas próximas versões do Gestor Express.

Para exportar um modelo no formato BPMN do Gestor Express prossiga para as etapas a seguir:

  1. Selecione um modelo de processo do item modelagem e clique duas vezes nele. Um pmodelo de rocesso também pode ser aberto clicando na opção Editar na barra de ferramentas superior.
  2. Uma vez que o modelo de processo é aberto no Mapa do Processo, selecione a opção Exportar Modelo.
  3. O modelo será baixado para o computador local com a extensão .bpmn
  4. A extensão .bpmn permite ao usuário importar o modelo para uma instalação/espaço de trabalho diferente. Para fazer isso, clique no botão Importar na aba modelagem (em uma instalação/espaço de trabalho diferente do Gestor Express).
  5. Clique no botão Upload para fazer upload do arquivo para a instalação/espaço de trabalho do Gestor Express.

    O modelo será aberto no Mapa de Processos conforme a imagem abaixo.

  6. Observe que se já existir um processo com o mesmo nome, será exibida uma mensagem perguntando ao usuário se deseja substituir o processo.

Se a opção Substituir estiver selecionada, o modelo importado substituirá todo o conteúdo do processo existente. Isso significa que qualquer formulário dinâmico, documento de entrada, documento de saída, gatilho, mensagem, tabela de relatório, conexão de banco de dados, variável ou supervisor será excluído, mesmo que o modelo seja igual ao importado.

Como usar um modelo de processo importado

Antes de usar um modelo de processo importado, atribua usuários a cada tarefa. Lembre-se de que as contas de usuário não são importadas com um modelo de processo, portanto, é necessário criar contas de usuário válidas. Clique com o botão direito do mouse em cada tarefa no mapa do processo para atribuir esses usuários.

Em alguns processos, um grupo de usuários já está atribuído a uma tarefa. Nesse caso, você precisará atribuir seus próprios usuários a esse grupo. Na guia usuários, crie contas de usuário e atribua-as aos grupos apropriados. Depois que os usuários válidos forem adicionados às tarefas ou grupos, um processo poderá ser executado. Faça login como o usuário atribuído à primeira tarefa e inicie um caso. Em seguida, faça login com as contas de usuário atribuídas às tarefas subsequentes e continue a executar o processo do lado do usuário.

Na lista abaixo, encontre elementos e configurações do Gestor Express que são importados com sucesso quando um processo está sendo movido de um espaço de trabalho/instalação para outro:

Elementos importados:

  • Formulário Dinâmicos
  • Documentos de entrada
  • Documentos de saída
  • Gatilhos
  • Atribuição de grupo em cada tarefa
  • Conexões de banco de dados
  • Configuração do cronograma de casos
  • Configuração do evento
  • Configuração da tarefa
  • Modelos de e-mail

Elementos não importados:

  • Usuários
  • Permissões de processo
  • Supervisores de Processo
  • Categoria de Processo
  • Tabelas PM (Precisam ser exportadas e importadas separadamente)
  • Estrutura do departamento
  • Atribuição do usuário em cada tarefa
