Visão geral

Os usuários finais interagem com um formulário por meio dos controles disponíveis nele. Os controles do designer Formulário Dinâmico responsivo HTML5 usam variáveis globais criadas dentro do projeto para interagir com elementos do projeto quando os dados necessários devem ser inseridos por um usuário por meio de um formulário. O Gestor Express criou uma interface que permite aos usuários definir essas variáveis, adicionar opções e definir o tipo de dados.

Controles Gerais

Quando um novo formulário é criado, o contêiner onde todos os controles serão adicionados (formulário) e o contêiner onde cada controle específico será adicionado também (linha) têm suas próprias funcionalidades e propriedades específicas dentro deles.

  • Formulário
  • Linha

Controles da Web

O designer Formulário Dinâmico responsivo vem com os seguintes controles que estão diretamente relacionados às variáveis criadas no projeto:

  • Caixa de texto e área de texto
  • Data hora
  • Suspenso
  • Caixa de seleção e grupo de seleção
  • Rádio (grupo)
  • Sugerir
  • Oculto

Os seguintes controles que estavam disponíveis nas versões anteriores do Gestor Express não estão disponíveis na versão 3:

  • Caixa de listagem
  • Sim/Não
  • Percentagem
  • Moeda

Se precisar gerenciar os controles acima, leve em consideração que as propriedades dos novos controles substituem a funcionalidade da maioria deles simplesmente configurando o comportamento dos controles.

Outros controles

Esses controles estão disponíveis dentro da Ferramente de Desenho e dão mais funcionalidades aos formulários como gerenciamento de documentos, links, imagens, adição de código HTML extra, etc. Esses controles não estão diretamente relacionados a uma variável do projeto. Esses controles são adicionados diretamente ao projeto e cada um deles pode ser configurado usando suas propriedades. Os controles disponíveis desta seção são:

Valor Nulo

A seguir estão as condições usadas para trabalhar com valores nulllário Dinâmicos

  • Campos de Formulário Dinâmicos relacionados a variáveis possuem inicialmente um valor null (o valor nulo é interpretado como "valor ausente").
  • Se o campo (renderizado) não for alterado pelo usuário em um Formulário Dinâmico em nenhum momento, o valor continuará sendo null.
  • Uma string vazia é diferente do valor null. Se o usuário limpar o conteúdo de um campo, a string vazia é usada como valor válido (significa que o valor salvo será a string vazia).
  • Leve em consideração que as variáveis armazenam valores nulos, portanto, os campos Tabelas de Relatórios e Tabelas GE também devem suportar valores nulos

Valor padrão

A seguir estão as condições usadas para trabalhar com valores padrão no Formulário Dinâmicos

  • O valor padrão em um campo de um formulário dinâmico é definido como o valor recuperado do campo (quando é renderizado) se ele não contiver nenhum valor anterior.
  • O valor padrão é definido como o valor recuperado do campo somente se seu valor for null(não o valor definido pelo usuário, mesmo que esse valor seja uma string vazia).
  • O valor padrão substitui as propriedades "SQL" e "Opções". Esta é a ordem em que os valores são salvos (mostrado por prioridade):
    • Primeiro valor SQL//prioridade menor
    • Valor da primeira opção
    • Valor padrão//prioridade máxima
  • Se o valor padrão for definido, ele substituirá as opções ou valores SQL. Se o valor padrão não for definido, o primeiro valor na propriedade Opções substituirá o valor definido pelo SQL.

Valor padrão em campos com valores de domínio

O valor padrão deve ser definido dentro do domínio de valores recuperados pela consulta SQL ou definido nas Opções da variável do campo. A seguir estão os campos em que esta consideração deve ser levada em conta:

  • Suspenso
  • Grupo de verificação
  • Grupo de rádio

Esses campos procurarão o valor definido como padrão nos dados retornados pela consulta SQL ou pelas opções definidas para o controle. Se o valor for encontrado nos dados recuperados, o valor padrão será definido como o valor por padrão. Se o valor padrão não for encontrado nesses domínios, o valor padrão não será salvo nos dados do controle.

Valor padrão para sugestões

No caso do campo de sugestão, o comportamento do valor padrão é diferente. O valor padrão pode estar fora do domínio definido no domínio (SQL e opções). Não é necessário que o valor padrão faça parte do domínio.

O valor padrão será exibido na área de texto do controle de sugestão e armazenado na variável. Nesse caso, o valor e o texto exibidos são os mesmos.

Valor padrão em campos de valor único

Nos demais controles que não definem um domínio de opções, basta definir o valor padrão nas propriedades do controle.

Definindo vários valores padrão

Os grupos de verificação permitem zero, uma ou várias opções sejam selecionadas por padrão. Para selecionar várias opções, separe os valores das opções selecionadas com brras verticais (|). Lembre-se de usar os valores das opções, e não os rótulos, que é o texto exibido ao usuário. Por exemplo, se desejar selecionar apenas o Canadá em um grupo de verificação de países, insira CA na propriedade de valor padrão. Para selecionar Bolívia, Argentina e Estados Unidos por padrão, insira BO|AR|US na propriedade de valor padrão.

Campos que devem selecionar um valor inicial

Os campos que definem um domínio de opções definem um grupo de valores possíveis.

Somente o controle suspenso sempre seleciona um valor quando é renderizado pela primeira vez. Para definir um valor vazio ou nulo, crie uma opção sem valor e adicione qualquer descrição como “--Selecione uma opção--”.

Observação: Foi adicionada a propriedade Placeholder neste controle no qual é possível definir um valor vazio.

Se a lista suspensa for obrigatória e o valor retornado for nulo, a validação retornará falsa e não será possível ir para a próxima etapa. O formulário pedirá para selecionar um valor na lista suspensa.

Formato SQL

A propriedade SQL é usada em muitos controles no Formulário Dinâmico. Leve em consideração as seguintes condições para definir as instruções SQL. Para obter mais informações sobre a propriedade SQL, consulte Propriedades do controle Formulário Dinâmico - SQL.

Instrução SQL que retorna valores e texto

As seguintes regras devem ser aplicadas aos controles:

  • Caixas Suspensa
  • Grupo de verificação
  • Grupo de rádio
  • Sugestão

A seguinte sintaxe deve ser usada na propriedade SQL:

SELECT [coluna de valor], [coluna de texto] FROM [nome da tabela] WHERE [qualquer condição]

Onde:

  • Coluna de valor: É o identificador da opção.

    • Observação: Os identificadores suportam apenas texto de uma palavra e caracteres como . e @. Os seguintes caracteres não são suportados quando estão entre palavras: ', |. `,  (espaço), ", ! e ^ (por exemplo: A^A). Além disso, a Ferramenta de Desenho do Formulário Dinâmico do Gestor Express não oferece suporte a consultas SQL com o identificador *. Usar o identificador * é considerado uma prática ruim porque gera problemas de desempenho.

  • Coluna de texto: É o texto que é exibido no controle.
  • Nome da tabela: O nome da tabela no banco de dados.
  • condição: Qualquer condição para a consulta SQL

Instrução SQL que retorna um valor

As seguintes regras devem ser aplicadas aos controles:

  • Caixa de texto
  • Área de texto
  • Ocultar

A seguinte sintaxe deve ser usada na propriedade SQL:

SELECT [coluna de valor] FROM [nome da tabela] WHERE [qualquer condição]

Onde:

  • Coluna de valor: O valor deve ser sempre o primeiro campo da consulta. É o identificador da opção.

    • Observação: O identificador * não é suportado em consultas SQL.

  • Nome da tabela: O nome da tabela no banco de dados.
  • condição: Qualquer condição para a consulta SQL.

SQL e opções em controles

Existem dois grupos de controles que usam as propriedades SQL e Options de maneira diferente. Ambos são explicados abaixo

SQL e opções que definem o domínio de opções

Os valores retornados pela consulta SQL e as propriedades Options constituem os valores de domínio do campo. Os valores válidos para o controle são apenas os definidos no domínio. Os seguintes controles podem conter e usar a propriedade SQL e a propriedade Options como valores de domínio:

  • Suspenso
  • Grupo de verificação
  • Grupo de rádio
  • Sugestão

Observação: O identificador * não é suportado em consultas SQL.

Propriedade SQL em controles escalares

A propriedade SQL também pode ser usada como um valor escalar em um controle. Se houver mais de um valor como resultado da execução da consulta SQL, apenas o primeiro valor será utilizado para definir o valor de controle.

Os seguintes controles podem conter e usar a propriedade SQL como valor escalar:

  • Caixa de texto
  • Area de Texto
  • Oculto

Considere as regras definidas na seção “Valor padrão” para determinar qual valor é definido quando o SQL e o Valor padrão são definidos. Observe que esses controles não possuem a propriedade Options.

Observação: O identificador * não é suportado em consultas SQL.

Variáveis Duplicadas em Formulários Dinâmicos

Dentro de um Formulário Dinâmico é recomendado definir apenas uma instância de uma variável. No entanto, é possível definir dois campos usando a mesma variável nas seguintes situações:

  • A mesma variável é atribuída em dois ou mais subformulários na Ferramenta de Desenho do formulário (significa que os subformulários adicionados usaram a mesma variável em seus controles). Lembre-se de que um formulário só pode ser usado uma vez como subformulário em um formulário, mas um formulário pode ter mais de um subformulário em seu modelo.
  • O modelo do subformulário é modificado depois que o subformulário foi atribuído a vários formulários. E uma variável é duplicada em qualquer uma das formas principais.

O desenhista deseja duplicar o uso da variável em duas seções do formulário e exibir ou ocultar um dos campos associados dependendo do modelo. Nesse caso, a Ferramenta de Desenho do Formulário Dinâmico vai alertar o desenhista que uma variável está sendo utilizada em dois ou mais campos, mas o desenhista deve definir se o campo referente à mesma variável é possível.

Ao executar casos, se uma variável for usada em dois ou mais controles, o controle no qual o valor da variável é definido no final será salvo como o valor da variável. A definição da ordem dos controles é de cima para baixo e da esquerda para a direita (observe que isso depende do navegador). Para evitar esses problemas é recomendado não duplicar variáveis.

Para saber mais sobre restrições de variáveis, consulte as melhores práticas de nomenclatura de variáveis.
