Visão geral

O Gestor Express oferece dois tipos diferentes de controles para inserir texto diretamente em um formulário. Estes são os controles Caixa de Texto e Área de Texto.

Controle de caixa de texto

Esse controle adiciona um campo de caixa de texto onde os usuários podem inserir texto digitando ou colando. Para adicionar esse controle à Ferramenta de Desenho do Formulário Dinâmico, arraste o ícone abaixo do painel Controles da Web à esquerda para uma das linhas no modelo do Formulário Dinâmico.

Gerenciando Caixas de Texto

Após adicionar uma caixa de texto ao modelo de Formulário Dinâmico, é possível personalizar sua funcionalidade e comportamento, utilizando o painel de propriedades. Clique no controle para exibir suas propriedades no painel à esquerda da Ferramenta de Desenho do Formulário Dinâmico.

As propriedades definidas por padrão são as seguintes:

  • tipo: definido por padrão como "text". Esta definição não pode ser modificada.
  • id: Definido por padrão como textVar00X, onde "X" representa a numeração correspondente do controle no modelo. Esta numeração começa em 1. Após uma variável ser relacionada a este controle, o ID muda para o nome da variável, porém é possível definir um novo ID para trabalhar com o controle.
  • rótulo: Definido por padrão como "text_X" onde "X" representa a numeração correspondente do controle no modelo. Esta numeração começa em 1.
  • Transformar texto para: Definido por padrão como o "nenhum" que significa que o texto será salvo da maneira que o usuário final o inserir.
  • Comprimento Máximo: Definido por padrão com o número 1000 que será o número máximo de caracteres que um usuário poderá inserir na caixa de texto.
  • modo de exibição: definido por padrão como "mestre". Consulte esta documentação para obter mais informações sobre os modos disponíveis para este controle.
  • Conexão de banco de dados: definido por padrão como "PM Database". Leve em consideração que se nenhuma consulta SQL for definida na propriedade "sql", nenhuma execução será feita. Além disso, quando uma variável está relacionada a este controle que possui sua própria definição da conexão com um banco de dados (um banco de dados externo ou banco de dados do Gestor Express) esta propriedade herda a definição da variável relacionada no controle.

Transformar texto em

Esta opção transforma qualquer texto inserido automaticamente em UPPER CASE, lower case, Capitalize phrase(a primeira letra é alterada para maiúscula) ou Title Case(a primeira letra de cada palavra é alterada para maiúscula). Selecione o tipo de transformação dentro das propriedades do controle.

As alterações serão mostradas quando o formulário for renderizado dentro da visualização ou ao executar um caso. Por exemplo:

Máx. Comprimento

Esta propriedade define o número máximo de caracteres que podem ser inseridos em um campo de controle de texto. Esta propriedade também é uma validação feita sobre estes campos, mas se uma expressão regular foi definida dentro da propriedade "Validate", esta validação será verificada antes do Max. Comprimento.

Teoricamente, a definição da propriedade permite que um máximo de 9.999.999.999 caracteres sejam inseridos no campo, mas o navegador da Web provavelmente não terá memória suficiente para lidar com essa quantidade de entrada. Além disso, o Gestor Express só pode armazenar até 16 MB de dados sobre um caso, que seriam 16.777.216 caracteres (supondo que não haja caracteres internacionais que precisem de mais de 8 bits). Em uma nota prática, é melhor usar Área de Textos e objetos de arquivo para armazenar grandes quantidades de dados, pois eles são armazenados separadamente no banco de dados e não retardam o processamento do Gestor Express, nem estão sujeitos às mesmas limitações de espaço que as caixas de texto.

Fórmula

Nota: Regras ou máscaras de validação podem ser usadas nos campos de texto envolvidos em operações matemáticas para restringir a entrada de dados.

Esta propriedade permite que fórmulas matemáticas sejam utilizadas com base nos campos do Formulário Dinâmico. As operações matemáticas suportadas são adição, subtração, multiplicação e divisão. As fórmulas também podem usar (...)(parênteses) para agrupar operações. Os campos usados em uma fórmula já devem existir no Formulário Dinâmico. A fórmula deve ser colocada no campo onde será exibido o resultado.

Clique no botão "Editar" na propriedade "Fórmula" do controle de texto:

Neste exemplo, o campo "Resultado" mostrará o resultado da multiplicação entre "Valor 1" e "Valor 2". No entanto, leve em consideração que a fórmula deve ser feita usando o ID desses controles de texto, então a fórmula a ser inserida seria:

Ao executar um caso, os campos que configuraram uma fórmula são desabilitados para entrada de texto. Os resultados das operações matemáticas são mostrados à medida que os valores são inseridos nos campos incluídos na fórmula.

As fórmulas podem consistir nos IDs dos campos de texto, números e os seguintes operadores matemáticos:

OperadorExemplo

+salesPrice + tax

-salesPrice - discount

*salesPrice * quantity

*pounds / 2.2046226

(...)(currentSales - previousSales) / previousSales

A multiplicação e a divisão têm maior precedência do que a adição e a subtração. Na seguinte fórmula:

preço - desconto * taxRate

Primeiro, o discounté multiplicado por taxRate, então o resultado é subtraído de price. Parênteses podem ser usados para alterar a ordem das operações matemáticas:

(preço - desconto) * taxRate

Agora, o discounté subtraído do price, então o resultado é multiplicado pelo taxRate.

Os resultados do cálculo são exibidos no campo que contém a fórmula no momento em que o usuário altera o valor de um dos campos utilizados na fórmula. Se o usuário inserir um valor em um campo que não pode ser interpretado como um número, o campo que contém a fórmula ficará em branco (ou seja, definido como uma string vazia). Para forçar o usuário a inserir apenas números válidos nos campos, é recomendável usar uma máscara ou definir a propriedade validate para aceitar apenas números:

Apenas números inteiros:^\d+$

Apenas números inteiros positivos ou negativos:^[\-+]?\d+$

Números inteiros ou decimais:^[\-+]?\d*\.?\d*$

Número com até dois dígitos decimais:^[\-+]?\d*(\.\d{0,2})?$

Número com exatamente dois dígitos decimais:^[\-+]?\d*\.\d{2}$

Nota: Para que as fórmulas funcionem corretamente, os números inseridos pelo usuário nos campos NÃO devem ter separador de milhar e devem usar.(ponto) como separador decimal, como: 1234567.89

Se precisar usar números como 123,456,789.01(vírgula como separador de milhar), 123 456 789.01(espaço como separador de milhares) ou 123.456.789,01(ponto como separador de milhares e vírgula como separador decimal) e, em seguida, adicione JavaScript personalizado que usa control.setOnchange() para fazer o cálculo.

As fórmulas também podem ser usadas em campos de texto encontrados em grades. As fórmulas nas grades podem conter apenas os IDs de outros campos na mesma linha da grade, portanto, não é possível fazer referência a campos fora da grade ou campos em outras linhas da grade. Se precisar fazer cálculos com base em campos fora da grade, é recomendável adicionar código JavaScript que usa o formulário.setOnchange() para definir um manipulador de eventos que calcula a fórmula quando um valor na grade é alterado.

Validar

Esta propriedade permite que seja inserida uma expressão regular que verifica se o usuário inseriu uma entrada válida na caixa de texto. Se a entrada corresponder à expressão regular, o usuário poderá sair do campo, mas se a entrada não corresponder, a mensagem do campo será destacada em vermelho e o texto na propriedade Mensagem de erro de validação será exibido. O usuário não terá permissão para sair do campo até que sua entrada seja alterada para corresponder à expressão regular.

Sintaxe de Expressão Regular

Esta é uma breve introdução às expressões regulares que podem ser usadas na propriedade validate para verificar se o usuário inseriu uma entrada válida em uma caixa de texto.

Correspondência de sequências de caracteres

Para corresponder a qualquer sequência de caracteres, use-os na expressão regular. Por exemplo, uma propriedade validate de thaceitaria qualquer entrada que contenha as letras th, como thumb, e. No entanto, ele não aceitaria porque as expressões regulares diferenciam maiúsculas de minúsculas.somethingSmithTom Thumb

Início do texto:^

Observe que a expressão regular pode ocorrer em qualquer lugar do texto. Para especificar que a expressão regular deve ocorrer no início do texto, preceda-a por ^. Por exemplo, uma propriedade validate de ^Smaceitaria Smithe Smothers, mas não aceitaria ou porque não ocorre no início do texto.John SmithSSSSmeeegol!Sm

Fim do texto:$

Da mesma forma, para especificar que a expressão regular deve ocorrer no final do texto, adicione $após a sequência de caracteres para corresponder. Por exemplo, uma propriedade validate de 2X$aceitaria e, mas não aceitaria ou porque não é encontrada no final do texto.Size 2X42X2X largerUA2XFR2X

Para corresponder ao texto completo, use ambos ^antes da sequência de caracteres para corresponder e $no final da sequência de caracteres. Por exemplo, ^yes$só permitiria que o usuário entrasse yesno campo. Não corresponderia yesou Tell me yes.

Caracteres de escape:\

Se estiver procurando por um texto que contenha ^ou $qualquer um dos outros caracteres especiais usados por expressões regulares, coloque uma \(barra invertida) na frente deles, que escapará deles para que sejam tratados como literais. Por exemplo, a expressão regular \$10aceitaria o texto the price is $10.99. Na expressão regular ^4\^3, a string de caracteres 4^3deve ocorrer no início do texto.

Qualquer caractere:.

Use um.(ponto) para corresponder a qualquer caractere, exceto uma nova linha. Por exemplo, UX..0correspondências e. Se precisar pesquisar por um ponto, escape-o com um precedente (barra invertida). Por exemplo, corresponderia e. UX500-200Serial YUX_R034-200MUX %0\0\.00.045%230.00

Conjunto de caracteres:[xyz]

Para corresponder a qualquer caractere em um conjunto de caracteres, coloque os caracteres dentro [](colchetes). Por exemplo, corresponderia c[oal]a e, mas não corresponderia a, porque não está no conjunto de caracteres. copcatblood clotcrewr

Também não corresponderá cLopporque o conjunto de caracteres diferencia maiúsculas de minúsculas. Para não diferenciar maiúsculas de minúsculas, use [cC][oalOAL], para que corresponda a Cop, cAtou CLEAN.

Para corresponder a um intervalo de caracteres, use -para especificar qualquer caractere entre dois caracteres. Por exemplo, use [ad-fm]para corresponder aos caracteres a, d, eou. A expressão regular corresponde a qualquer letra minúscula ou maiúscula ou a qualquer número. As letras que não estão no conjunto de caracteres ASCII não são incluídas no intervalo, portanto, lembre-se de incluí-las se os usuários estiverem trabalhando com idiomas que tenham outras letras. Por exemplo, inclui todas as letras em espanhol.fm[a-zA-Z0-9][a-z][a-záéíóúüñA-ZÁÉÍÓÚÜÑ]

Dentro [](os colchetes), todos os caracteres como $*.?()são tratados como literais e não têm um significado especial. Para incluir -, [, ]ou \no conjunto de caracteres, escape-os com um precedente \(barra invertida). Por exemplo, [!@#$%*(){}\[\]_\-+=/\\|<>~]inclui a maioria dos símbolos padrão.

Conjunto de caracteres negado:[^xyz]

Para corresponder a qualquer caractere que não esteja em um conjunto de caracteres, coloque os caracteres dentro [^ ](colchete aberto, acento circunflexo, colchete fechado). Por exemplo, g[^aeiou]match gseguido por qualquer caractere que não seja uma vogal, como growe. bog!

Ocorre 0 ou 1 vezes:?

Se a expressão anterior não puder ocorrer ou ocorrer uma vez, coloque um ?(ponto de interrogação) depois dela. Por exemplo, tr?eecorresponde a tee-shirte. Se colocado após um conjunto de caracteres, qualquer um dos caracteres no conjunto pode ocorrer zero ou uma vez. Por exemplo, correspondências e.oak treec[rl]?awbear clawsticks in my crawcrow's cawing

Ocorre 0, 1 ou mais vezes:*

Se a expressão anterior puder ser zerada, uma ou mais vezes, coloque um *(asterisco) depois dela. Por exemplo, me*tcorrespondências e. Se colocado após um conjunto de caracteres, qualquer um dos caracteres no conjunto pode ocorrer zero, uma ou mais vezes. Por exemplo, correspondências e.metmeetingmt Everest\$[0-9]*$0price: $2346give me $ now

Ocorre 1 ou mais vezes:+

Se a expressão anterior puder ocorrer uma ou mais vezes, coloque um +(sinal de mais) depois dela. Por exemplo, me+tcorresponde a mete meeting, mas não corresponde mt Everestporque o etem que ocorrer pelo menos uma vez entre me t. Da mesma forma, \$[0-9]+corresponde $0e, mas não corresponde porque deve haver pelo menos um número após o cifrão.price: $2346give me $ now

Ocorre um número fixo de vezes:{n}

Se a expressão anterior ocorrer um número fixo de vezes, especifique o número dentro {}(chaves). Por exemplo, bo{2}gcorresponderia a boogyman, mas não bogeymanporque o precisa ocorrer duas vezes seguidas. A expressão regular ^[0-9]{4}$só aceitaria um inteiro positivo com 4 dígitos, como 2406ou 0987. Da mesma forma, ^[-+]?[0-9]{4}\.[0-9]{2}$só aceitaria números positivos ou negativos com 4 dígitos e 2 dígitos decimais, como 2324.99ou -0876.08.

Ocorre um intervalo de vezes:{n, m}

Se a expressão anterior ocorrer entre n e m número de vezes, especifique o intervalo inferior e o intervalo superior dentro {}(chaves). Por exemplo, bo{2,4}gcorresponderia a boogymane, mas não e não. A expressão regular só aceitaria um número inteiro positivo entre 1 e 3 dígitos, como, ou. Da mesma forma, só aceitaria números positivos ou negativos com entre 1 e 4 dígitos e entre 1 e 3 dígitos decimais, como ou.It's a booooger!wet bogfour boooooooogies!^[0-9]{1,3}$387097^[-+]?[0-9]{1,4}\.?[0-9]{0,3}$4.999-0876

A faixa inferior ou a faixa superior não precisa ser especificada. Por exemplo, ^[0-9]{2,}$aceitaria quaisquer inteiros positivos com 2 ou mais dígitos. Da mesma forma, ^[0-9]{,5}$aceitaria qualquer número inteiro positivo com 5 ou menos dígitos.

Agrupamento:(xyz)

Para agrupar uma sequência de caracteres, coloque-os ()entre parênteses. O agrupamento pode ser especialmente útil quando combinado com ?, *, +ou {}. Por exemplo, R(IP)*corresponderia a, e. A expressão regular corresponderia a e, mas não corresponderia porque não ocorre duas vezes seguidas.GRIPRIPIPIPIPIPPPPTurn Right(99)+23499378.999992392399

Ou:x|y

Para corresponder a uma ou outra expressão, separe-as por |(uma barra vertical). Por exemplo, cat|dogcorrespondências e. Não há limite para o número de strings alternativas de caracteres que podem ser correspondidas, então muitos tipos de animais de estimação podem ser combinados com. Se as strings alternativas forem encontradas em uma string maior de caracteres, coloque as strings alternativas entre parênteses. Por exemplo, corresponderia e.I love catsbad doggie!cat|dog|fish|bunny|snake|ferret()bad (check|contract)s?It's a bad checkbad contracts kill business

Limite de palavra:\b

para pesquisar um limite de palavra, use \b. Uma palavra consiste apenas em letras, números e sublinhados [a-zA-Z0-9_]. Quando uma palavra está ao lado de qualquer outro caractere ou no final ou no início do texto, ela é considerada um limite de palavra. As linhas verticais azuis no texto a seguir são consideradas limites de palavras: | Pode | ' | t | | dizer | " | Olá |," | pode | | você | ?

Os limites das palavras podem ser úteis ao pesquisar palavras separadas. Por exemplo, uma pesquisa por catcorresponderia a cattle, caterpillere, mas corresponderia apenas ao texto que contém a palavra separada "gato", como e. Uma pesquisa mais flexível para "gato" permitiria que a palavra fosse maiúscula ou pluralizada:hellcat\bcat\bGive me the cat.cat\b[Cc][aA][aT][sS]?\b

Nota: As palavras não incluem letras não ASCII, como ü, é, ou ñ, portanto, não é muito útil ao lidar com outros idiomas além do inglês. Uma boa maneira de contornar esse problema é usar um conjunto de caracteres para o idioma. Por exemplo, pesquisaria a palavra separada gato no idioma espanhol.φあ(^|[^a-záéíóúüñA-ZÁÉÍÓÚÜÑ])gato($|[^a-záéíóúüñA-ZÁÉÍÓÚÜÑ])

Sem limite de palavra:\B

Para procurar uma string que não ocorra próximo a um limite de palavra, use \B(observe o B maiúsculo). Por exemplo, \Bmancorresponderia a e, mas não corresponderia a e.cattlemanhumanehelp the manmanhole

Os limites das palavras podem ser úteis ao pesquisar palavras separadas. Por exemplo, uma pesquisa por catcorresponderia a cattle, caterpillere, mas corresponderia apenas ao texto que contém a palavra separada "gato", como e. Uma pesquisa mais flexível para "gato" permitiria que a palavra fosse maiúscula ou pluralizada:hellcat\bcat\bGive me the cat.cat's gut\b[Cc][aA][aT][sS]?\b

Não ganancioso:?

muitas vezes é útil usar expressões como.*que corresponde a qualquer número de caracteres ou [a-zA-Z]*que corresponde a qualquer número de letras, mas essas expressões não funcionam corretamente se for necessário pesquisar o texto que segue depois. Por exemplo, serial.*UX48não corresponderia serial RX-UX48, pois.*continuaria correspondendo a todos os caracteres até o final do texto, que nunca veria a UX48parte. Isso se chama ser ganancioso. Para dizer à expressão regular para não ser gananciosa e verificar se o texto que está sendo consumido.*corresponde ao que segue depois, adicione ?(um ponto de interrogação). A expressão regular serial.*?UX48é capaz de corresponder serial RX-UX48conforme o esperado.

Exemplo de expressões regulares

O Gestor Express fornece alguns exemplos de expressões regulares que podem ser inseridas na propriedade validate. Para ver a lista de exemplos, clique no botão Ajuda. Em seguida, clique em um dos exemplos da lista para inseri-lo e depois personalizá-lo para atender às suas necessidades.

Abre-se uma janela com os exemplos de expressões regulares que podem ser utilizadas nesta propriedade. Para usar qualquer um deles basta clicar na opção necessária.

A expressão regular selecionada será inserida no valor da propriedade onde poderão ser feitas as edições correspondentes para personalizar a validação.

Como na imagem acima, a expressão regular utilizada para validar um endereço de e-mail fornecido pela opção de ajuda desta propriedade é a seguinte:

^[a-zA-Z0-9]+.?[a-zA-Z0-9]+@[a-zA-Z0-9]+.[a-zA-Z0-9]{2,4}$

Essa expressão regular verifica se a primeira seção do endereço de e-mail contém letras maiúsculas ou minúsculas e/ou números de 0 a 9 na expressão [a-zA-Z0-9]. Em seguida, a expressão verifica se o endereço de e-mail é seguido de um caractere especial opcionalmente e, novamente, letras e/ou números após ele na expressão.?[a-zA-Z0-9]. Como essa validação é feita para um endereço de e-mail, a validação também verifica se o endereço contém o caractere "@" seguido novamente por caracteres que contêm letras e/ou números, um ponto e depois mais letras e/ou números.

Se estiver usando a validação mostrada anteriormente, ao renderizar o formulário a validação será mostrada conforme a imagem abaixo ao inserir um endereço de e-mail válido.

Para um endereço de e-mail inválido, o aviso será mostrado quando o evento OnChange() for acionado.

Por exemplo, o código a seguir é usado para verificar a entrada em uma caixa de texto com o ID "productCode". Quando o foco sai do campo, o código remove automaticamente quaisquer espaços e converte a entrada em maiúsculas. $ ("#productCode"). getControl (). change (function (e) {

val = this. value. trim () ; // remove os espaços iniciais e finais

val = val. toUpperCase () ;

re = /^S[0-9][0-9][XYT] [0-9]$/ ;

if (val. match (re)) {

//configura o valor transformado no campo "serial":

$ ("#productCode"). setValue (val) ;

}

else {

alert (this. value + " é inválido. \n Use números de série como 'S[0-9][0-9][XYT][0-9]'.") ;

}

}) ;

Filtrando chaves com JavaScript

A propriedade validate pode ser útil para verificar a entrada em um campo, mas só acontece durante um evento de alteração quando o foco sai do campo. Para evitar que o usuário insira uma entrada inválida em um campo, pode ser útil filtrar pressionamentos de tecla para caracteres inválidos retornando falseno manipulador de eventos para os eventos keydown, keypress ou keyup.
