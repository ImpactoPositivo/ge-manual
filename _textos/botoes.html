Visão geral

Os botões são um objeto de interface que pode ser clicado pelos usuários para executar uma ação. Se for um botão enviar, o formulário dinâmico é enviado, significando que o formulário é fechado e os dados inseridos nos campos são armazenados em variáveis ​​de caso e salvos no banco de dados. O caso continuará para a próxima etapa na tarefa atual ou roteará para a próxima tarefa se for a última etapa. Se for um botão genérico, o código JavaScript personalizado pode ser adicionado ao formulário dinâmico, que será executado quando o botão for clicado.

Propriedades do botão Enviar

As propriedades associadas a este controle são:

PropriedadeDescrição

JavaScript em botões

Para saber como manipular controles Formulário Dinâmico usando JavaScript, consulte JavaScript em Formulários Dinâmicos.

Botões genéricos e botões de envio no Gestor Express têm os seguintes componentes de campo:

  • Rótulo: O texto no botão.
  • Valor: o valor é o mesmo do rótulo.
  • Texto: O texto é o mesmo do rótulo.

Métodos JavaScript

Alguns dos métodos JavaScript para manipular botões genéricos e botões de envio incluem:

MétodoDescrição
jQuery("#campoID").getValue()Retorna o texto do botão.
jQuery("#campoID").getText()getText() atualmente tem um bug e não funciona com controles de botão.
jQuery("#campoID").getrótulo()Retorna o texto do botão.
jQuery("#campoID").setValue("novoTexto")Define o texto do botão.
jQuery("#campoID").setText("novoTexto")Define o texto do botão.
jQuery("#campoID").setrótulo("novoTexto")Define o texto do botão.

Para ver exemplos de código para esses métodos, consulte Funções e métodos JavaScript.

Observação: chamar setText(), setrótulo() ou setValue() para um botão genérico ou botão de envio redefinirá suas propriedades e removerá quaisquer propriedades personalizadas definidas pelo JavaScript. Após chamar setText(), setrótulo() ou setValue(), essas propriedades precisarão ser redefinidas.

Observação: getControl() não funciona com botões, mas um botão pode ser obtido usando o método .find(). Por exemplo:

$("#submitForm").find("button").css("border","1px solid red");

A propriedade "id" pode ser usada com código JavaScript. Para isso, clique na borda cinza claro do formulário dinâmico (que é o espaço fora dos controles) para exibir as propriedades do formulário dinâmico. jQuery("#submit0000000001").find("button").setValue("www.google.com");

E o resultado da adição desse código alterará o nome do valor do controle "enviar".

A propriedade "rótulo" alterará o rótulo do controle. Se o código JavaScript foi adicionado e o valor deseja ser definido manualmente. Apague o código Javascript e altere o rótulo com um nome como "ENVIAR" como visto na imagem abaixo.

Uma nova tela rápida ao enviar um caso aparecerá, não permitindo que os usuários cliquem no botão "Enviar" mais de uma vez, em vez de várias vezes, o que pode resultar em avançar um passo no processo e não concluir algumas etapas. Agora, o botão "Enviar" só pode ser clicado uma vez, portanto, indo para a tarefa a seguir, conforme o esperado. Alterar a cor do campo

Como exemplo de como usar o controle de botão, adicione uma caixa de texto e um controle de botão na ferramenta de desenho do formulário. Em seguida, altere o ID dos controles para "texto1" e "botão123", respectivamente.

Agora, adicione o seguinte JavaScript na propriedade JavaScript do formulário.

$("#botao123").find("button").click(function(){
   $("#texto1")[0].style.backgroundColor = "blue";
  if($("#texto1")[0].style.display == "none")
    $("#texto1").show();
  else
    $("#texto1").hide();
});

Em seguida, experimente a funcionalidade do botão renderizando-o na opção Visualizar

Cancelar caso

Um formulário dinâmico pode conter vários botões de envio com finalidades diferentes. Este exemplo mostra como criar um formulário dinâmico que tem um botão chamado "Enviar" que continua o caso e outro botão chamado "Cancelar caso", que cancela o caso atual. O código JavaScript é usado para atribuir um valor a um campo oculto quando o botão de envio "Cancelar caso" é clicado. Em seguida, um gatilho disparado após o Formulário Dinâmico verificar esse valor para determinar se deve chamar Case::cancelCase().

Primeiro, crie um formulário dinâmico com um botão de envio normal e um segundo botão de envio com o ID "cancelarCaso". Além disso, adicione um campo oculto cujo ID seja "ação" e esteja associado à variável acao.

Em seguida, adicione o seguinte código JavaScript para atribuir o valor "CANCELLED" ao botão "cancelarCaso" se clicado:

$("#cancelarCaso").find("button").click(function(){
  $("#acao").setValue("CANCEL");
});

Em seguida, crie o seguinte gatilho que é definido para disparar após o formulário dinâmico:

if (@@acao == "CANCEL") {
   $c = novo Casos();
   $c->cancelarCaso(@@APPLICATION, @%INDEX, @@USER_LOGGED);
   //redireciona de volta para a caixa de entrada:
   echo '<script> top.location = "../cases/main" </script>';
   die();
};

O gatilho acima verifica se o botão "CancelarCaso" foi clicado quando o formulário foi enviado. Se sim, cancela o caso. Em seguida, ele redireciona o navegador da Web de volta para a caixa de entrada. Observe que os casos são executados dentro de um quadro, portanto, o Javascript é usado para redirecionar o navegador da Web para o quadro superior. A função die() é chamada para parar o Gestor Express e evitar que o caso passe para a próxima etapa do processo.

Observação: As variáveis de caso não são mais criadas para botões de envio, portanto, esses exemplos foram alterados para atribuir um valor a campos ocultos, que podem ser examinados em um acionador subsequente.

Redirecionar para outra etapa

Quando um botão de envio é clicado, os dados no formulário dinâmico são salvos e o caso passa para a próxima etapa (formulário dinâmico, documento de entrada ou documento de saída) na tarefa atual. Se for a última etapa, o Gestor Express roteará para a próxima tarefa no processo. O código JavaScript pode ser adicionado a um botão para redirecionar para uma etapa diferente na tarefa atual ou para o final da tarefa quando o botão for clicado.

Este exemplo mostra como redirecionar o navegador da Web quando os botões são clicados. Um formulário dinâmico que é o primeiro passo da tarefa tem os seguintes 6 botões:

  • Um botão com o ID "salvaTrabalho" que salva os dados no formulário dinâmico atual quando clicado.
  • Um botão com o ID "caixaDeEntrada" que redireciona o navegador de volta para a caixa de entrada.
  • Um botão com o ID "fimTarefa", que direciona o navegador para o final da tarefa atual.
  • Um botão com o ID "maisInfo" que redireciona o navegador web para a 3ª etapa da tarefa, que é um Formulário Dinâmico com o UID "140382730582d22af0d1c24018284816" para inserir mais informações.
  • Um botão com o ID "uploadContrato" que redireciona o navegador para a quarta etapa da tarefa, que é um Documento de Entrada com o UID "77782079956a7de25a125c5065741006" para fazer upload de arquivos de contrato.
  • Um botão com o ID "geraNota" que redireciona o navegador web para a quinta etapa da tarefa, que é um Documento de Saída com o UID "13321001556d6125088d942072818002".
  • Um botão de envio normal, que continuará para a próxima etapa da tarefa (ou roteará para a próxima tarefa se for a última etapa da tarefa atual).

O seguinte código JavaScript é adicionado ao Formulário Dinâmico para adicionar manipuladores de eventos aos botões quando eles são clicados:

$("#salvaTrabalho").find("button").click(function(){
   $("form").saveForm();
});
$("#caixaDeEntrada").find("button").click(function(){
   $("form").saveForm();
   top.location="../cases/main"; //redireciona para a caixa de entrada no quadro superior
});
$("#fimTarefa").find("button").click(function(){
   $("form").saveForm();
   //redireciona para o final da tarefa para roteamento
   location="../cases/cases_Step?TYPE=ASSIGN_TASK&UID=-1&POSITION=10000&ACTION=ASSIGN";
});
$("#maisInfo").find("button").click(function(){
   $("form").saveForm();
   //redireciona para um Formulário dinâmico que é a 3ª etapa no local atual da tarefa
   location="../cases/cases_Step?TYPE=DYNAFORM&UID=140382730582d22af0d1c24018284816&POSITION=3&ACTION=EDIT";
});
$("#uploadContrato").find("button").click(function(){
   $("form").saveForm();
   //redireciona para o documento de entrada que é a 4ª etapa no local da tarefa atual
   location="../cases/cases_Step?TYPE=INPUT_DOCUMENT&UID=77782079956a7de25a125c5065741006&POSITION=4&ACTION=ATTACH";
});
$("#geraNota").find("button").click(function(){
   $("form").saveForm();
   //redirecionar para o documento de saída que é a 5ª etapa no local da tarefa atual
   location="../cases/cases_Step?TYPE=OUTPUT_DOCUMENT&UID=13321001556d6125088d942072818002&POSITION=5&ACTION=GENERATE";
});

Para entender melhor como o redirecionamento funciona no Gestor Express, consulte a documentação de Links e a função G::header().

Observação: Não é possível redirecionar para um formulário dinâmico, documento de entrada ou documento de saída que não esteja atribuído como uma etapa na tarefa atual. Também não é possível redirecionar para uma etapa específica em outra tarefa.

Redirecionamento em gatilhos

Aviso: Os exemplos a seguir NÃO funcionam na versão do Gestor Express Mobile.

O exemplo anterior chama o método form.saveForm() para salvar os dados atuais inseridos no formulário dinâmico quando um botão normal é clicado. No entanto, um botão de envio salvará automaticamente os dados do Formulário Dinâmico quando clicado, portanto, não é necessário usar a função saveForm(). Se precisar salvar os dados do formulário e depois fechar o Formulário Dinâmico, é recomendável usar botões de envio em vez de botões normais. Além disso, adicione um campo oculto ao formulário dinâmico cujo ID seja "ação" e esteja associado a uma variável denominada "ação". Este campo oculto indicará qual botão foi clicado para fechar o formulário.

Em seguida, crie um gatilho que use PMFRedirectToStep() para redirecionar para uma etapa (Formulário Dinâmico, Documento de entrada ou Documento de saída) ou use o método G::header() para redirecionar para a Caixa de entrada ou o final da tarefa para roteamento.

O seguinte código JavaScript pode ser adicionado ao formulário dinâmico para indicar qual botão de envio foi clicado:

$("#saveWork").find("button").click(function() {
   //salva o trabalho e volta para o mesmo Formulário Dinâmico
   $("#action").setValue("SAVE_WORK");
});
$("#inbox").find("button").click(function() {
   $("#action").setValue("INBOX");
});
$("#endTask").find("button").click(function() {
   $("#action").setValue("END_TASK"); //ir para roteamento no final da tarefa
});
$("#moreInfo").find("button").click(function() {
   $("#action").setValue("MORE_INFO"); //ir para outro formulário dinâmico
});
$("#uploadContract").find("button").click(function() {
   $("#action").setValue("UPLOAD_CONTRACT"); //ir para o documento de entrada
});
$("#generateInvoice").find("button").click(function() {
   $("#action").setValue("GENERATE_INVOICE");  //ir para o documento de saída
});

Em seguida, crie o seguinte gatilho para lidar com o redirecionamento do navegador da web:

if (!isset(@@action)) {
   goto triggerEnd;
}
$action = @@action;
PMFSendVariables(@@APPLICATION, array('action' => '')); //redefinir ação

if ($action == "INBOX") {
   echo '<script> top.location = "../cases/main" </script>';
   die(); //impedir próximo passo
}
elseif ($action == "END_TASK") {
   G::header("Location: cases_Step?TYPE=ASSIGN_TASK&UID=-1&POSITION=10000&ACTION=ASSIGN");
   die(); //impede a próxima etapa
}
elseif ($action == "SAVE_WORK") {
   //redireciona de volta ao Formulário Dinâmico anterior para continuar trabalhando nele:  
   PMFRedirectToStep(@@APPLICATION, @%INDEX, 'DYNAFORM', '60422430056a69386a2a885087313352');
}
elseif ($action == "MORE_INFO") {  
   PMFRedirectToStep(@@APPLICATION, @%INDEX, 'DYNAFORM', '140382730582d22af0d1c24018284816');
}
elseif ($action == "UPLOAD_CONTRACT") {
   PMFRedirectToStep(@@APPLICATION, @%INDEX, 'INPUT_DOCUMENT', '77782079956a7de25a125c5065741006');
}
elseif ($action == "GENERATE_INVOICE") {
   PMFRedirectToStep(@@APPLICATION, @%INDEX, 'OUTPUT_DOCUMENT', '13321001556d6125088d942072818002');
}
triggerEnd:

Esse gatilho deve ser definido para ser acionado antes da próxima etapa que segue o formulário dinâmico (ou antes da atribuição, se for a última etapa da tarefa). NÃO defina esse gatilho para disparar imediatamente após o formulário dinâmico, pois chamar die() nesse ponto impedirá que os dados nos campos do formulário dinâmico sejam adicionados ao campo wf_WORKSPACE.APPLICATION.APP_DATA no banco de dados e impedirá que quaisquer arquivos selecionados sejam salvos no sistema de arquivos do servidor.
