Visão geral

Um pool é usado para delimitar um processo separado, que geralmente vem de uma organização diferente. Todos os tipos de fluxos podem conectar processos normais a subprocessos, mas processos em conjuntos separados podem se comunicar apenas por meio de fluxos de mensagens.

Entendendo piscinas e raias

Um pool é um limite organizacional que engloba todos os elementos de um único processo. Um pool é usado para delimitar o processo de uma organização e criar comunicação com os processos de outras organizações. Ao contrário dos fluxos de sequência normais dentro do Gestor Express, que são mostrados como linhas pretas sólidas, os fluxos de mensagens entre os pools são mostrados como linhas pretas tracejadas.

Existem dois tipos de piscinas. Um pool normal (também conhecido como pool de caixa branca) exibe seus detalhes internos, de modo que é possível criar conexões diretamente com os objetos dentro do pool. Em contraste, um pool de caixa preta oculta todos os seus detalhes internos, portanto, não é possível visualizar os eventos, tarefas, gateways etc. que constituem o processo dentro de um pool de caixa preta. Somente as conexões de e para um pool de caixa preta podem ser visualizadas.

Dentro de uma piscina normal, os elementos em um processo podem ser divididos usando linhas horizontais ou verticais, que são chamadas de raias, ou mais simplesmente, raias. Na maioria dos casos, essas raias são usadas para separar as tarefas atribuídas a diferentes usuários ou grupos, como gerenciamento, vendas, contabilidade, etc., mas as raias podem ser usadas para qualquer tipo de agrupamento, como tempo, localização geográfica, organização hierárquica, etc

Piscinas

Para adicionar uma piscina ( que chmremos de pool, de acordo com nomenclatura internacional) ao modelo de processo, arraste e solte o símbolo da pisicna da caixa de ferramentas de formas no mapa do modelador.

A nova paiscina será automaticamente denominada "Pool 1" quando adicionado ao Mapa do Processo.

Os pools têm uma ordem crescente, portanto, ao adicionar um novo pool, ele começará com "Pool 1", em seguida será "Pool 2", depois "Pool 3" e assim por diante.

Para alterar o nome do pool, clique duas vezes em seu nome para ativar o modo de edição.

Para adicionar elementos ao pool, como atividades, eventos e gateways, mova os elementos existentes para o pool ou arraste novos elementos da caixa de ferramentas Formas e solte-os dentro do pool.

Lembre-se que os elementos são facilmente adicionados usando a Barra de Ferramentas Rápida.

Elementos que não fazem parte da caixa de pool são sobrepostos pela piscina e seus elementos, mesmo que pareçam fazer parte dos elementos da caixa de pool. Quando uma caixa de pool é movida, ela se move junto com seus elementos, mas os elementos que não fazem parte da caixa de pool não se movem.

Observação: para adicionar um processo existente a um pool, NÃO crie um novo pool sobre o processo. Em vez disso, crie o novo pool em uma área em branco no modelador e redimensione-o para que seja grande o suficiente para abranger todo o processo. Em seguida, selecione todo o processo com a ferramenta seletora de grupo e arraste-o para o pool.

Para mover um pool, clique nele para que os quadrados verdes apareçam em suas bordas, depois arraste e solte-o em um novo local no mapa do modelo.

Para redimensionar um pool, clique nele para que os quadrados verdes apareçam em suas bordas. Em seguida, clique em um dos quadrados verdes e arraste-a para redimensionar a caixa.

Para excluir um pool, clique nele para que os quadrados verdes apareçam em suas bordas. Em seguida, pressione a tecla Delete no teclado. Todos os elementos dentro do pool também serão excluídos.

Outra maneira de excluir o pool é clicar nele e o ícone da lixeira aparecerá (disponível a partir da versão 3.0.1)

Raias ou lanes

As raias são usadas para dividir uma piscina em seções. Eles podem ser usados para indicar diferentes departamentos ou escritórios, regiões geográficas, diferenças conceituais, etc.

Para adicionar uma raia a uma piscina, arraste uma raia da caixa de ferramentas de formas e solte-a dentro de uma piscina existente.

A nova raia aparecerá na piscina, adicionando um rótulo no lado esquerdo da piscina.

Atualmente, há uma inconveniência ao criar pools que impede que elementos existentes em um pool (como eventos, atividades e gateways) sejam editados após a adição de uma raia. Quando isso acontecer, uma mensagem de erro rápida aparecerá. Portanto, é recomendável primeiro adicionar as raias a uma piscina e adicionar os elementos posteriormente.

Para adicionar seções adicionais à piscina, arraste mais raias da caixa de ferramentas Formas e solte-as na piscina.

As raias podem ser renomeadas e elementos adicionais podem ser adicionados às raias.

Cada raia pode ser redimensionada clicando nela para fazer com que quadrados verdes apareçam nas bordas inferior e direita. Em seguida, arraste e solte a caixa verde em um novo local para redimensionar a raia.

Para redimensionar ou mover a piscina inteira após adicionar raias, clique no lado esquerdo da piscina onde seu nome está localizado para fazer com que quadrados verdes apareçam ao redor de toda a piscina. Em seguida, arraste e solte um dos quadrados verdes para redimensioná-la. Para movê-lo, clique para baixo dentro da piscina e arraste e solte-o em um novo local.

Para excluir uma raia e todos os novos elementos que foram adicionados dentro dela, clique na raia para que uma caixa verde apareça em suas bordas e, em seguida, pressione a tecla Delete no teclado.

Uma raia também pode ser apagada clicando com o botão direito nela e selecionando Excluir. A raia desaparecerá imediatamente.

Fluxos de mensagens entre piscinas (pools)

De acordo com o padrão BPMN 2.0, cada pool deve conter um processo separado, com eventos de início e término e um fluxo ininterrupto do início ao fim. Para se comunicar entre processos em conjuntos separados, os fluxos de mensagens podem passar entre os processos, mas os fluxos de sequência não podem passar entre os conjuntos. Os fluxos de sequência são representados por linhas sólidas, enquanto os fluxos de mensagens são representados por linhas tracejadas.

Para enviar uma mensagem entre pools, clique em um elemento dentro do primeiro pool e, quando a Barra de Ferramentas Rápida aparecer, use uma seta de conexão para arrastar uma linha do primeiro pool para o segundo pool.

Observe que o fluxo é indicado com uma linha tracejada, o que significa que é um fluxo de mensagens, em vez de um fluxo de sequência.

Piscinas de Caixa Preta

Observação: leve em consideração que o Gestor Express 3.0 em diante implementou Black Box Pools no designer de processo, mas esses pools ainda não foram implementados no mecanismo de processo. Portanto, os Black Box Pools devem ser usados apenas para fins de design. Por favor, fique atento a lançamentos futuros.

Adicione um Black Box Pool arrastando e soltando seu símbolo da caixa de ferramentas Shapes no mapa do processo.

Adicione um rótulo a um pool de caixa preta clicando duas vezes no pool e inserindo o texto do rótulo.

Para mover uma piscina, clique uma vez na piscina, de modo que fique delimitada por quadrados verdes. Em seguida, arraste e solte-o em um novo local. Observe que nenhum outro elemento pode ser colocado dentro de um pool de caixa preta, portanto, não é uma boa ideia colocá-lo em cima de qualquer elemento existente no mapa do processo. Para redimensionar uma piscina, use os quadrados verdes nas laterais para ajustar a piscina ao tamanho necessário.

No momento, soltar elementos em um Black Box Pool não será eficaz, portanto, é aconselhável não adicionar nenhum elemento e usá-los apenas para fins ilustrativos.

Para excluir um pool, pressione a tecla Delete no teclado quando um pool estiver delimitado por quadrados verdes. Um Black Box Pool também pode ser apagado clicando nele e usando a Barra de Ferramentas Rápida.
