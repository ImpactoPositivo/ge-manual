Visão geral

Nos Formulários Dinâmicos do Gestor Express os controles Data e Hora permitem que o usuário final adicione datas a um Formulário Dinâmico por meio de um calendário interativo.

Clicar em um campo de controle Data e Hora abre um pop-up de calendário onde os usuários podem facilmente selecionar e inserir uma data. Isso ajuda a padronizar os formatos de data e evita erros comuns.

Os Formulários Dinâmicos no Gestor Express usam a biblioteca Bootstrap 3 Datepicker para construir seu controle Data e Hora. Esta biblioteca baseia seu formato na biblioteca Moment.js, uma poderosa biblioteca JavaScript que gerencia datas e formatos. As seções a seguir abordarão como definir as propriedades do controle Data e Hora para saber mais sobre como ele funciona.

Gerenciando um controle de data e hora

Depois de adicionar um controle de data e hora à Ferramenta de Desenho do Formulário Dinâmico, é possível personalizar sua funcionalidade e comportamento usando o painel de propriedades.

As propriedades definidas por padrão são as seguintes:

  • tipo: definido por padrão como "datetime". Esta definição não pode ser modificada.
  • id: Definido por padrão como datetimeVar00X, onde "X" representa o número correspondente do controle no desenho. Por exemplo, se o controle Data e Hora fosse o primeiro controle Data e Hora a ser adicionado ao Formulário Dinâmico, X seria 1, se fosse o segundo controle Data e Hora a ser adicionado ao Formulário Dinâmico, X seria 2 e assim por diante. Depois que uma variável é relacionada a esse controle, o ID muda para o nome da variável; no entanto, é possível definir um novo ID para o controle usando JavaScript.
  • rótulo: Definido por padrão como "datetime_1", onde "X" representa o número correspondente do controle no desenho. Esta numeração começa em 1.
  • modo de exibição: Definido por padrão como "mestre". Consulte esta documentação para obter mais informações sobre os modos disponíveis para esse controle.
  • formatar: Definido por padrão como " YYYY-MM-DD", que representa o formato de data "Ano-Mês-Dia". Leia a explicação desta propriedade abaixo para saber como configurar o formato.
  • data de seleção inicial: Defina o valor do campo e do seletor de data quando o seletor de data for aberto. Esta propriedade é definida como "falso" por padrão. Para obter mais informações, consulte esta seção.
  • modo de visualização: Definido por padrão como "dias" para mostrar um calendário do mês em que o selecionador de data é aberto.
  • mostrar botão limpar: Definido por padrão como "esconder" para ocultar o ícone de lixo no seletor de data ao trabalhar com o campo no Formulário Dinâmico. Para mais informações, leia a explicação desta propriedade abaixo.

Além disso, um ícone será adicionado na parte inferior da caixa de diálogo do seletor de data se o formato incluir horas, minutos, segundos ou AM/PM. Clicar nele mudará a caixa de diálogo para selecionar as unidades de tempo no formato:

Sobre hh:mm

Se o formato a seguir for definido nesta propriedade:

hh:mm

Observe que o seletor de horas NÃO pode diferenciar entre "am" e "pm" por si só. Portanto, se o usuário selecionar 17:00 com esta configuração, será escrito como 05:00 devido ao formato do hh token. Para evitar ambiguidade e diferenciar entre "am" e "pm", adicione a definição de meridiano:

hh:mm A

Observação: os controles de data e hora permitem que o usuário selecione as horas e os minutos na caixa de diálogo do seletor de data se a propriedade de formato do controle contiver horas e minutos. No entanto, pode ser mais conveniente simplesmente criar uma caixa de texto normal onde o usuário possa inserir o valor de data e hora. Para fazer isso, defina sua propriedade validação para:

^(19[0-9]{2}|20[0-9]{2})-(1[0-2]|0[1-9])-(0[1-9]|[12][0-9]|3[01])( [0-5][0-9]:[0-5][0-9]){0,1}$

Devido à biblioteca Date não fornecer controle total sobre o formato hh:mm, para exibir corretamente o seletor de data com o formato 12h, defina a propriedade formatar adicionando um espaço antes de hh:mm. A propriedade formatar também permite exibir valores de controle Data e Hora adicionando um espaço anterior.

Data de seleção inicial

A propriedade Data de Seleção Inicial define a data e hora padrão no campo de controle e o selecionador de data. Esta propriedade é definida como "falsa" por padrão.

As outras opções nesta propriedade são as seguintes:

  • Falso: O valor do campo e do seletor de data permanecerá vazio após a abertura do seletor de data.
  • Verdadeiro: A data e hora atuais serão definidas no seletor de data e no campo quando o seletor de data for aberto. Por exemplo, 28-04-2017 11:26
  • Ano: O ano da data atual será definido. O dia e o mês serão definidos como "01" e a hora como "00:00". Por exemplo, 01-01-2017 00:00.
  • Mês: O ano e o mês da data atual serão definidos. O dia será definido como "01" e a hora como "00:00". Por exemplo, 01-04-2017 00:00
  • Dia: O ano, mês e dia da data atual serão definidos. A hora será definida como "00:00". Por exemplo, 28-04-2017 00:00
  • Hora: O ano, mês, dia e hora da data e hora atual serão definidos com os minutos como "00". Por exemplo, 28-04-2017 11:00
  • Minuto: O ano, mês, dia, hora e minuto da data e hora atual serão definidos. Por exemplo, 28-04-2017 11:26

Observe que é necessário configurar o formato para mostrar horas e minutos no campo Data e Hora e no selecionador de data.

Data padrão

Essa propriedade define a data e hora padrão ao usar o seletor de data.

Esta propriedade não é configurada por padrão. Para definir uma data padrão, insira-a manualmente (atenção ao formato) ou, para evitar erros, clique no calendário à direita da caixa do campo de controle e selecione a data padrão no seletor de data e hora.

Modo de visualização do selecionador de data

Esta propriedade define a visualização padrão do calendário exibido quando o seletor é mostrado. Os valores possíveis deste controle estão descritos abaixo.

  • "Dias" Mostra os dias do mês no calendário do selecionador de datas.
  • "Meses" Mostra os meses do ano no calendário do selecionador de datas.
  • "Anos" Mostra os anos no calendário do selecionador de datas.

Essa propriedade é definida como "Dias" por padrão.

Mostrar botão Limpar

Esta propriedade mostra o botão Limpar na barra de ferramentas do ícone (representado por um ícone de lixeira). Quando este botão é clicado, o campo de data e hora e o valor do selecionador de data são limpos.

JavaScript em controles de data e hora

Para saber como manipular os controles do Formulário Dinâmico usando JavaScript, consulte JavaScript em Formulários Dinâmicos.

Os controles de data e hora no Gestor Express têm os seguintes componentes de campo:

  • Rótulo: O texto exibido acima ou à esquerda do campo em um formulário dinâmico renderizado para identificá-lo ao usuário.
  • Texto: A data e hora exibida para o usuário, conforme configurado pela propriedade de formato.
  • Valor: O valor armazenado do campo no formato padrão "AAAA-MM-DD" para datas e formato "AAAA-MM-DD hh:mm:ss" para datas e horas. Exemplos: "2010-01-01", "1999-07-25 04:01:01", "2015-12-31 23:59:59"

Obtendo datas

O valor dos controles Data e Hora é armazenado como uma string no formato "YYYY-MM-DD HH:mm:ss", como "1999-12-31 23:59:59" ou "2017-01-15 08:25:03". Use o método getValue() para obter o valor de um controle Data e Hora. O valor sempre usa o mesmo formato.

Em contraste, o método getText() retornará a data formatada exibida ao usuário. O que é exibido no campo de controle Data e Hora é definido por sua propriedade formatar. Por exemplo, um Data e Hora usando o formato "MM/DD/YY" pode ter um valor de "2017-08-26 00:00:00" e exibir o texto "08/26/17".

Se as horas, minutos e/ou segundos não estiverem incluídos na propriedade formatar de data e hora , eles serão automaticamente definidos como zero. Por exemplo, um Data e Hora com o formato "DM-YYYY h:m a" pode ter o valor "1980-07-04 16:15:00" e exibir o texto "4-7-1980 4:15 pm". Observe que os segundos são definidos como "00" porque não estão incluídos no formato. Por exemplo, para obter o valor e o texto exibido de um controle Data e Hora cujo ID é "dueDate":

var dueValue = $("#dueDate").getValue();

var dueText = $("#dueDate").getText();

Definir datas e horas

O método setValue() é usado para definir uma data e hora. Certifique-se de que a data e hora esteja no formato "YYYY-MM-DD HH:mm:ss". As horas, minutos e segundos são opcionais e serão definidos como 00 por padrão, se não forem incluídos. Por exemplo, para definir os valores dos controles Data e Hora que possuem os IDs "startDate", "endDate" e "pauseDate":

$("#datainicial").setValue("2017-06-23");

$("#endDate").setValue("2017-11-01 17:30");

$("#pauseDate").setValue("2017-10-25 08:15:50");

O método setText() também pode ser usado para definir o valor de um controle Data e Hora. O novo Data e Hora deve estar no formato padrão "YYYY-MM-DD HH:mm:ss" ou no formato personalizado definido pela propriedade de controle formatar.

Comparando Data e Horas em JavaScript

Para manipular datas em JavaScript, as strings de data e hora precisam ser convertidas em Data. Para facilitar isso, o Gestor Express integrou a biblioteca JavaScript moment.js.

Moment.js simplifica a manipulação relacionada a data e hora e é compatível com Gestor Express Desktop, Gestor Express Mobile para iOS e Gestor Express Mobile para Android. Leia a documentação oficial do moment.js para saber mais sobre as funções desta biblioteca.

Exemplo-Tempo entre a data atual e a data selecionada

O código abaixo recupera o tempo decorrido entre a data atual e a data selecionada pelo usuário.

function checkTime(){

         var startDt = $('#datetimeVar').getValue();

         var rest = moment(startDt).fromNow();

         $("#textVar").setValue(rest);

};

$('#button0000000004').click(checkTime);

function comparandoDatas(){

Acessando Data e Horas com PHP

Quando um formulário dinâmico é enviado, suas Data e Horas serão armazenadas como strings nas variáveis associadas ao case, que podem ser manipuladas com código PHP em um gatilho. Por exemplo, se uma Data e Hora de um Formulário Dinâmico estiver associado à variável orderDate, ele poderá ser acessado em um gatilho como a variável de caso @@orderDate. Se um gatilho definir a variável de caso @@orderDate antecipadamente, o controle Data e Hora associado a essa variável exibirá seu valor quando o Formulário Dinâmico for exibido.

Por exemplo, para definir o valor de um controle Data e Hora associado à variável orderDate para a data atual, crie um gatilho que seja acionado antes do Formulário Dinâmico e use a função getCurrentDate() do Gestor Express para definir a data:

@@orderDate = getCurrentDate();

Da mesma forma, o valor inserido em um campo de dados do Formulário Dinâmico pode ser lido em um gatilho acionado após o envio do Formulário Dinâmico:

$x = @@ orderDate ;

Além de getCurrentDate(), o Gestor Express fornece a função personalizada getCurrentTime() para retornar a hora atual como uma string. Se a opção for selecionada para usar também a hora (HH:mm:ss) em um campo de data, uma data e sua hora podem ser definidas concatenando as strings retornadas pelas duas funções:

@@orderDate = getCurrentDate(). " ". getCurrentTime();

O valor de um Data e Hora é armazenado como uma string na variável de caso associada do controle no formato padrão "YYYY-MM-DD HH:mm:ss", como "2017-01-25 23:59:59". Se a propriedade de formato do controle Data e Hora não contiver horas, minutos e/ou segundos, esses elementos serão definidos como 00, como "2017-01-25 00:00:00". Este é o mesmo formato usado pelo banco de dados MySQL para armazenar datas e horas.

Comparando Data e Horas com PHP

Se estiver fazendo comparações simples de Data e Horas, como testar se uma data é anterior , posterior ou igual a outra data, as duas datas podem ser comparadas como strings. Por exemplo:

if (@@DueDate < @@StartDate ou @@DueDate >= "2010-12-31") {

...

}

Convertendo Data e Horas em Timestamps

Para fazer operações mais avançadas com Data e Horas, é necessário primeiro converter as Data e Horas em um timestamp PHP, que é o número de segundos entre o Data e Hora atual e 1 de janeiro de 1970 00:00:00 GMT. Use a função PHP strtotime() para converter um Data e Hora de uma string para um timestamp. Por exemplo, para converter @@orderDate em timestamp:

$x = strtotime(@@orderDate);

A função strtotime() pode converter qualquer data armazenada para o formato padrão "AAAA-MM-DD". Se estiver trabalhando com outros tipos de strings de data, reordene a string de data para um dos formatos abaixo com os quais strtotime() funciona:

Formato Exemplo de strtotime()
YYYY Mês DD strtotime("2000 Março 15")
DD Mês YYYY strtotime("15 Março 2000")
Mês DD, YYYY strtotime("Março 15, 2000")
Seg DD, YYYY strtotime("Mar 15, 2000")
MM/DD/YYYY strtotime("03/15/2000")
MM/DD/YY strtotime("03/15/00")
YYYY/MM/DD strtotime("2000/03/15")
DD-MM-YYYY strtotime("15-03-2000")
YY-MM-DD strtotime("00-03-15")
YYYY-MM-DD strtotime("2000-03-15")

Se separar os elementos com uma "/" (barra), strtotime() espera uma data no estilo inglês, enquanto um "-" (hífen) indica uma data no estilo internacional. strtotime() também aceita datas relativas, significando o tempo decorrido entre a hora selecionada e a hora atual. strtotime() aceita números positivos ou negativos seguidos pela unidade de tempo (segundos, minutos, horas, dias, semanas, meses, anos ):

$x = strtotime("+3 horas");

$x = strtotime("-15 semanas");

$x = strtotime("+1 semana 2 dias 4 horas 2 segundos");

Se a data deve ser relativa a uma hora diferente da atual, forneça um carimbo de data/hora como o segundo parâmetro:

$x = strtotime("+36 dias 7 horas", strtotime("2005-12-31"));

Depois de converter em timestamps, as datas podem ser comparadas.

Exemplo - Cobrar 15% a mais por qualquer entrega após a data de vencimento

Se estiver projetando um processo que cobra 15% a mais por qualquer entrega após a data de vencimento, use strtotime() para converter a data de entrega e a data de vencimento em timestamps e verifique se a data de entrega é posterior à data de vencimento:

if (strtotime(@@deliveryDate) > strtotime(@@dueDate)) {

     @%charge = @%charge + @%charge * 0.15;

}

Este código de gatilho enviará uma mensagem somente se a data de vencimento for hoje:

if (strtotime(@@dueDate) == strtotime(getCurrentDate()) {

     PMFSendMessage(@@APPLICATION , "boss@acme.com", "lazyemployee@acme.com", " ", " ",

     "Entregue HOJE!!!", "duetodaymessage.html");

}

Exemplo - Calcular a diferença entre duas datas em dias

Os Timestamps estão em segundos, portanto, subtrair um Timestamp de outro produzirá a diferença em segundos. Para obter a diferença em dias, divida por 24 horas vezes 60 minutos vezes 60 segundos:

$days = ($timestamp1 - $timestamp2) / (24*60*60)

O exemplo a seguir compara duas Data e Horas e retorna o número de dias entre eles. Depois disso, ele insere ambos as Data e Horas e o resultado em uma tabela PM chamada "PMT_MY_TABLE" usando a função executeQuery().

if (isset(@@date1) e !empty(@@date1) e isset(@@date2) e !empty(@@date2)) {

     $date1 = strtotime(@@date1);

     $date2 = strtotime(@@data2);

     $days = ($data2 - $data1) / (24*60*60);


       $sql = "INSERIR EM PMT_MY_TABLE (DATA1, DATA2, DIFERENCA_DIAS) VALORES ('$date1', '$date2',$days)";

     executeQuery($sql) ;

}

Para arredondar o número de dias, a função round() pode ser usada.

$days = round($days);

Para calcular o número de dias sem considerar a hora, a hora pode ser removida dos Data e Horas com a função substr().

if (isset(@@date1) and !empty(@@date1) and isset(@@date2) and !empty(@@date2)) {

     $date1 = strtotime ( substr(@@date1 , 0, 10) );

     $data2 = strtotime ( substr(@@date2 , 0, 10) );

     $dias = (int) (($data2 - $data1) / (24*60*60 ));

}

Exemplo - Calcular a diferença entre o primeiro dia do ano e o dia atual

O exemplo de código a seguir calcula o número de dias que se passaram desde o primeiro dia do ano atual até o dia atual:

$today = strtotime(date("Y-m-d"));

$newYear = strtotime(date("Y-01-01"));

$days = ($today - $newYear) / (24*60*60);

Exemplo - Calcular a diferença entre o dia atual e uma data e hora

O exemplo de código a seguir calcula a diferença entre o dia atual e um Data e Hora selecionado pelo usuário, em dias, horas, minutos e segundos.

if (isset(@@dueDate) and !empty(@@dueDate)) {

     $dueDate = strtotime(@@dueDate);

     $present = strtotime("now"); //present date and time

     $overdue = $dueDate - $present;

     $days = (int) ($overdue / (24*60*60));

     $remainder = $overdue % (24*60*60);

     $hours = (int) ($remainder / (60*60));

     $remainder = $remainder % (60*60);

     $minutes = (int) ($remainder / 60);

     $seconds = $remainder % 60;

     @@timeOverdue = empty($days) ?  ' ' : $days .' days';

     @@timeOverdue .= (empty(@@timeOverdue) ?  ' ' : '     ') . "$hours:$minutes:$seconds";

}

Extraindo informações de data e hora

A função getdate() do PHP é útil para extrair informações específicas de um timestamp. Getdate() retorna um array associativo com o número de "segundos", "minutos", "horas", "mday" (dia do mês), "wday" (dia da semana), "mon" (mês), "year" e "yday" (dia do ano), além de uma representação textual do "dia da semana" e do "mês". Por exemplo, para enviar uma mensagem se a data de vencimento for no mês atual:

$due = getdate(strtotime(@@dueDate));

$current = getdate();

if ($due["mon"] == $current["mon"]) {

    PMFSendMessage(@@APPLICATION, "chefe@acme.com,br", "colaborador@acme.com.br", " ", " ",

         "Entregar este mês", "mensagemvencimentodestemês.html");

}

Convertendo de Timestamps para Data e Horas

Depois de converter para um timestamp, use a função date() do PHP para converter um timestamp de volta para uma string, que pode ser exibida em um controle Data e Hora do Formulário Dinâmico ou inserido em um documento de saída ou modelo de e-mail. Para converter um timestamp para o formato "AAAA-MM-DD" ou "AAAA-MM-DD HH:mm:ss" usado pelo Gestor Express em controles de data e hora e no banco de dados MySQL, use:

date('Ymd', $timestamp)

ou

date('Ymd H:i:s', $timestamp)

Por exemplo, para adicionar 10 dias à data de vencimento a ser exibida em um Formulário Dinâmico subsequente:

$due = strtotime(@@dueDate);

$duePlusTen = strtotime("+10 days", $due);

@@dueDate = date("Y-m-d", $duePlusTen);

Formatação de data e hora em inglês

A função Date() também pode ser usada para criar representações de texto de data e hora. Por exemplo, para criar um Data e Hora em um formato como "terça-feira, 15 de março de 2010":

@@textualDueDate = date("l, F j, Y" , $duePlusTen);

Formatação de data e hora em outros idiomas

Para criar uma representação de texto de uma data em um idioma diferente do inglês, use a função strftime(). Ele irá formatar a data e hora no idioma da localidade onde o servidor Gestor Express está localizado. Para especificar uma localidade diferente, como "pt" (português), "pt_BR" (português do Brasil) ou "pt_BR@UTF-8" (português do Brasil no conjunto de caracteres UTF-8), use setlocale() do PHP. Por exemplo, para gerar a data de hoje em espanhol:

setlocale("LC_TIME" , "es");

@@textualDueDate = strftime("%A, %e %B %Y", time());

Para definir a localidade para o idioma do usuário atual do Gestor Express:

setlocale("LC_TIME" , @@LANG);

Observe que a localidade selecionada também deve ser instalada no servidor Gestor Express.

Para adicionar localidades no Debian/Ubuntu, use o comando como root:

dpkg-reconfigure locale

No Red Hat/CentOS/Fedora, vá para Sistema => Adicionar/Remover Pacotes. No Gerenciador de Pacotes, vá para a guia Browse e selecione Idiomas. Em seguida, clique no idioma desejado e instale-o.

Se vários idiomas estiverem instalados no Microsoft Windows, use setlocale() para selecionar o idioma. Consulte esta lista de localidades da Microsoft com suporte.

Data e Horas dinâmicos usando variáveis

As propriedades data padrão, data mínima e data máxima usadas por um controle Data e Hora podem ser definidas dinamicamente usando variáveis de caso. As propriedades do controle Data e Hora serão definidas pelas variáveis (desde que essas variáveis sejam definidas em um gatilho anterior ou em campos em um Formulário Dinâmico anterior). As variáveis devem usar o formato "YYYY-MM-DD" ou "YYYY-MM-DD HH:MM:SS", como "1999-01-01" ou "2016-12-31 23:59:59".

Exemplo - Configurando o controle para a data e hora atual:

Um controle Data e Hora cuja variável é denominada "currentDate" precisa ser definido para a Data e Hora atual. Primeiro, crie um gatilho definido para disparar antes do Formulário Dinâmico que usa as funções getCurrentDate() e getCurrentTime() para definir a data e hora atual no formato YYYY-MM-DD HH:MM:SS:

@@currentDate = getCurrentDate( ) . ' ' . getCurrentTime( );

Quando o Formulário Dinâmico for carregado, o controle Data e Hora associado à variável @@currentDate será preenchido automaticamente com o Data e Hora atual. Se o usuário não puder alterar o valor do campo Data e Hora, defina a propriedade valor protegido clicando no checkbox.

Exemplo - Definir três datas relativas

Neste exemplo, as variáveis @@minDate, @@maxDate e @@defaultDate são definidas nas propriedades data mínima, data máxima e data padrão, respectivamente.

O gatilho a seguir, que é executado antes do Formulário Dinâmico, define as três variáveis. Se o usuário definir a variável @@dueDate em um Formulário Dinâmico anterior, o gatilho definirá a data mínima para o primeiro dia do mês de @@dueDate, data máxima para o último dia do mês de @dueDate e a data padrão para o meio de o mês do vencimento. Se nenhum valor foi selecionado para @@dueDate, o gatilho definirá as três variáveis para o primeiro dia, último dia e dia do meio do próximo mês, respectivamente:

if (!isset(@@dueDate) or empty(@@dueDate)) {

    @@minDate = date('Y-m-01', strtotime('+1 month'));

    @@maxDate = date('Y-m-t', strtotime('+1 month'));

    @@defaultDate = date('Y-m-15', strtotime('+1 month'));

}

else {

    @@minDate = date('Y-m-01', strtotime(@@dueDate));

    @@maxDate = date('Y-m-t', strtotime(@@dueDate));

    @@defaultDate = date('Y-m-15', strtotime(@@dueDate));

}
