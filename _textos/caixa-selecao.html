Visão geral

A Ferramenta de Desenho do Formulário Dinâmico oferece dois controles diferentes para gerenciar opções com itens selecionáveis; os controles Caixa de seleção e grupo de seleção.

Caixa de seleção

Uma caixa de seleção oferece uma opção binária de ativação/desativação e aparece como uma caixa quadrada com uma marca de seleção quando marcada. As caixas de seleção são convenientes para permitir que um usuário selecione apenas uma opção com um único clique do mouse. Gerenciando Caixas de Seleção

Após adicionar uma caixa de seleção ao Formulário Dinâmico, é possível personalizar sua funcionalidade e comportamento usando o painel de propriedades.

As propriedades definidas por padrão são as seguintes:

  • tipo: Defina como " checkbox " por padrão. Esta propriedade não pode ser modificada.
  • id: definido como " checkbox 000000000X" por padrão, onde "X" representa o número correspondente do controle no projeto. Essa numeração começa em 1. Depois que uma variável é relacionada a esse controle, o ID muda para o nome da variável. No entanto, é possível definir um novo ID para o controle.
  • rótulo: Definido como " checkbox _X" por padrão, onde "X" representa o número correspondente do controle no desenho. Esta numeração começa em 1.
  • modo de exibição: definido como " parent" por padrão. Consulte esta documentação para obter mais informações sobre os modos de exibição disponíveis para este controle.

Acessando uma caixa de seleção com PHP

Quando uma caixa de seleção em um formulário dinâmico é enviada, sua variável associada tem um valor de array(1) se marcada ou array(0) se desmarcada. Por exemplo, o código de gatilho a seguir verifica se uma caixa de seleção associada à variável "temContrato" está marcada para definir o valor mínimo na variável @#minimaNota.

//se a caixa de seleção estiver marcada:

if (isset(@=temContrato) and @=temContrato == array(1)) {

    @#minimaNota = 1000.00;

}

else { //se desmarcado:

    @#minimaNota = 200.00;

}

Por padrão, a variável de rótulo para caixas de seleção é definida como "verdadeira", se marcada; ou "falso". se desmarcada; no entanto, esse valor pode ser personalizado na propriedade opções da caixa de seleção. Por exemplo, o gatilho acima pode ser escrito como:

//se a caixa de seleção estiver marcada:

if (isset(@@hasContract_label) and @@hasContract_label == 'true') {

    @#minimumInvoice = 1000.00;

}

else { //se desmarcado:

    @#minimumInvoice = 200.00;

}

Para definir o valor de uma caixa de seleção em um gatilho, defina sua variável associada com um valor de array(1) ou "verdadeiro" para marcá-la, ou um valor de array(0) ou "falso" para desmarcá-la. O gatilho a seguir marca uma caixa de seleção associada à variável temContrato em um gatilho disparado antes do formulário dinâmico que contém a caixa de seleção:

@=temContrato = array(1);

ou

@=temContrato = verdadeiro;

Grupo de verificação

Um grupo de verificação oferece uma opção sim/não para uma ou mais opções em um formulário dinâmico e aparece como uma ou várias caixas quadradas com uma marca de verificação quando selecionada (marcada). Os grupos de verificação são convenientes para permitir que um usuário selecione mais de uma opção com um único clique do mouse. Gerenciando Grupos de Verificação

Após adicionar um Grupo de Verificação ao modelo de Formulário, é possível customizar sua funcionalidade e comportamento configurando suas propriedades.

As propriedades definidas por padrão são as seguintes:

  • tipo: Defina como "checkgroup" por padrão. Esta propriedade não pode ser modificada.
  • id: definido como " checkgroupVar00X" por padrão, onde "X" representa o número correspondente do controle no projeto. Esta numeração inicia em 1. Após uma variável ser relacionada a este controle, o ID muda para o nome da variável; no entanto, é possível atribuir um novo ID ao controle.
  • rótulo: Definido como "checkgroup_X" por padrão, onde "X" representa o número correspondente do controle no desenho. Esta numeração começa em 1.
  • modo de exibição: definido como "mestre" por padrão. Consulte esta documentação para obter mais informações sobre os modos de exibição disponíveis para este controle.
  • fonte de dados : Definido por padrão como "banco de dados". Consulte esta documentação para obter mais informações sobre a fonte de dados disponível para esse controle.
  • Conexão de banco de dados: Defina como " PM Database" por padrão. Leve em consideração que se nenhuma consulta SQL for definida na propriedade sql, o banco de dados não será consultado. Além disso, quando uma variável está relacionada a um controle e possui sua própria conexão com um banco de dados (um banco de dados externo ou banco de dados do Gestor Express), essa propriedade herda a conexão da variável.

Observação: O identificador * não é suportado em consultas SQL.

Em seguida, as duas variáveis de caso a seguir serão adicionadas ao caso quando o Formulário Dinâmico que contém o grupo de verificação for enviado:

@= selectSubjects = array ( 'HIS-101' , 'FRE-302' , 'LIT-304' ) ;

@@ selectSubjects_label = '["Histórico 101","Francês 302","Literatura 304"]' ;

A variável de array do grupo de verificação precisa ser acessada como @=selectSubjects no código do gatilho. Se for acessado como @@selectSubjects, ele será convertido em uma string, o que causará problemas ao tentar usá-lo como um array.

Um item individual no array pode ser acessado como @=selectSubjects[ index ], onde o índice começa a contar a partir do número 0. A função count() pode ser usada para obter o número de itens selecionados no grupo de verificação. Se o usuário pode não ter enviado o formulário dinâmico ou pulado usando o menu Passos, então é uma boa ideia usar isset() para primeiro verificar se a variável do grupo de verificação foi definida. Verifique também se o número de itens selecionados é maior que o número do índice ao acessar o array.

Por exemplo, o código acionador a seguir obtém a chave da primeira opção selecionada no grupo de verificação selectSubjects :

if ( isset ( @= selectSubjects ) and count ( @= selectSubjects ) > 0 ) {

@@ firstSelectedKey = @= selectSubjects [ 0 ] ;

}

Um loop for ou foreach pode ser usado para acessar todas as opções selecionadas em um grupo de verificação.

Por exemplo, o código a seguir cria um novo caso no processo "Inscrever aluno" para cada item selecionado no grupo de verificação "selectSubjects". O código primeiro procura os IDs exclusivos do processo e sua primeira tarefa chamada "Verificar elegibilidade" e, em seguida, usa a função Derivation::getAllUsersFromAnyTask() para obter uma matriz dos usuários atribuídos à tarefa "Verificar elegibilidade". Em seguida, ele percorre as opções selecionadas no grupo de verificação "selectSubjects" e usa a função rand() para selecionar aleatoriamente um usuário para atribuir ao novo caso.

$processId = PMFGetProcessUidByName ( 'Inscrever Aluno' ) ;

$firstTaskId = PMFGetUidFromText ( 'Verificar Elegibilidade' , 'TAS_TITLE' , $processId , @@ SYS_LANG ) [ 0 ] ;

$d = nova Derivação ( ) ;

$aUsers = $d -> getAllUsersFromAnyTask ( $firstTaskId ) ;

if ( contagem ( $aUsers ) == 0) {

throw new Exception ( "Nenhum usuário atribuído à tarefa 'Verificar elegibilidade'." ) ;

}

foreach ( @= selectSubjects as $ subjectKey ) {

$random = rand ( 0 , count ( $aUsers ) -1 ) ; $assignedUserId = $aUsers [ $random ] ; $aVars = array ( 'subjectKey' => $subjectKey ) ;

$newCaseId = PMFNewCase ( $processId , $assignedUserId , $firstTaskId , $aVars ) ;

if ( vazio ( $newCaseId ) ) {

throw new Exception ( "Não foi possível criar um novo caso no processo 'Inscrever Aluno'." ) ;

}

}

Para acessar os rótulos das opções selecionadas no grupo de verificação, a função decode_json() pode ser usada para converter a variável @@selectSubjects_label em um array.

Por exemplo, o código de gatilho a seguir verifica se o usuário selecionou uma opção com o rótulo "Histórico 101" ou "Literatura 304" no grupo de verificação "selectSubjects". Nesse caso, a chave da opção selecionada é adicionada a uma variável de matriz chamada "languageRequirements".

@= languageRequirements = array ( ) ;

//verifica se o formulário dinâmico foi enviado:

if ( isset ( @@ selectSubjects_label ) e ! empty ( @@ selectSubjects_label ) ) {

$aLabels = json_decode ( @@ selectSubjects_label ) ;

for ( $i = 0 ; $i < contagem ( $aLabels ) ; $i ++ ) {

$label = $aLabels [ $i ] ;

$key = @= selectAssuntos [ $i ] ;

if ( $label == 'Histórico 101' ou $label == 'Literatura 304' ) {

@= languageRequirements [ ] = $key ;

}

}

}

Para definir as opções selecionadas em um grupo de verificação, adicione as chaves das opções à variável de matriz associada ao grupo de verificação. Certifique-se de que o gatilho esteja definido para disparar ANTES do formulário dinâmico que contém o grupo de verificação.

Por exemplo, o gatilho a seguir verifica se o aluno é formado em humanidades antes de marcar a primeira e a terceira opções no grupo de verificação "selectSubjects", que possuem as chaves 'HIS-101' e 'LIT-304':

if ( @@ major == 'Humanidades' ) {

@= selectSubjects = array ( 'HIS-101' , 'FRE-304' ) ;

}

Antes de adicionar uma opção à lista de opções selecionadas em um grupo de verificação, primeiro use a função in_array() para verificar se a opção já foi adicionada.

O código a seguir adiciona a opção 'FRE-302' à lista de opções selecionadas no grupo de verificação "selectSubjects":

//se não estiver definido, crie a variável do grupo de verificação:

if ( ! isset ( @= selectSubjects ) ) {

@= selectSubjects = array ( ) ;

}

//verifica se a opção ainda não está no array, antes de adicioná-la:

if ( ! in_array ( 'FRE-302' , @= selectSubjects ) ) {

@= selectSubjects [ ] = 'FRE-302' ;

}

A função unset() pode ser usada para remover uma opção da lista de opções selecionadas em um grupo de verificação. Use array_search() para primeiro encontrar o índice da opção selecionada.

Por exemplo, o seguinte código de gatilho desmarca a opção 'FRE-302' no grupo de verificação "selectSubjects":

if ( isset ( @= selectSubjects ) ) {

$index = array_search ( 'FRE-302' , @= selectSubjects ) ;

if ( $index !== false ) {

unset ( @= selectSubjects [ $ index ] ) ;

}

}

JavaScript em caixas de seleção e grupos de verificação

Para aprender a manipular controles Formulário Dinâmico usando JavaScript, consulte JavaScript em Formulários Dinâmicos .

As caixas de seleção/grupos de verificação no Gestor Express 3 têm os seguintes componentes de campo:

Rótulo: O texto exibido acima ou à esquerda do campo para identificá-lo ao usuário.

Texto: O texto (etiqueta) das opções do grupo de verificação, que podem ser configuradas na lista de opções. Este texto pode ser configurado em checkboxes, mas por padrão é "true" se marcado e "false" se desmarcado. Para grupos de verificação (e caixas de seleção anteriores à versão 3.0.1.4), o texto é uma string JSON contendo uma matriz dos rótulos das opções marcadas. Por exemplo, um grupo de verificação para selecionar países pode ter o seguinte texto:'["United States", "Mexico", "Brazil"]'

Valor: O valor armazenado (chave) da opção marcada. Para caixas de seleção, este é um valor de "1" se marcado e "0" se desmarcado. Para checkgroups, este é um array dos valores (chaves) das opções marcadas. Por exemplo, um grupo de verificação para selecionar países pode ter um valor de:["US", "MX", "BR"]

Caixas de seleção em grades

Obtendo valores de caixa de seleção em grades:

Na versão 3.1 e posterior, o método grid .getValue( row , column ) para uma caixa de seleção retorna uma string. Se a caixa de seleção na grade estiver marcada, ela retornará "1". Se desmarcado, ele retorna "0".

Por outro lado, na versão 3.0.1.8 e anteriores, o método grid .getValue( row , column ) para uma caixa de seleção retorna se marcado e se desmarcado. Se estiver atualizando o Gestor Express, o código deve ser alterado para levar essa alteração em consideração. Lembre-se também de que os números de linha e coluna nas grades começam a contar a partir do número . ["1"]["0"]1

Por exemplo, para verificar se uma caixa de seleção na segunda linha na terceira coluna de uma grade com o ID "clientList" está marcada na versão 3.1 e posterior:

if ( $ ( " # clientList " ) .getValue ( 2,3 ) == "1" ) { alert ( " Está verificado" ) ; }

Da mesma forma, o valor de uma caixa de seleção será "1"ou "0"na matriz retornada por grid.getValue(). Lembre-se de começar a contar as linhas e colunas do número 0ao acessar o array. Por exemplo, para verificar se uma caixa de seleção na segunda linha da terceira coluna de uma grade com o ID "clientList" está marcada:

if ( $ ( "# clientList " ) .getValue ( ) [ 1 ] [ 2 ] == "1" ) { alert ( "Está verificado" ) ; }

Uma caixa de seleção em uma grade pode ser marcada com ou grid .setValue(["1"], linha , coluna ). Para desmarcar uma caixa de seleção, use um valor de ou . setOnchange() com caixas de seleção

O método control .setOnchange() na versão 3.0.1.8 informa o valor de uma caixa de seleção como '["0"]'(desmarcada) ou '["1"]'(marcada). Na versão 3.1 e posterior, o control.setOnchange()método relata o valor de uma caixa de seleção como '"0"'(desmarcada) ou '"1"'(marcada).

Exibindo opções em uma lista horizontal

As opções em um grupo de verificação são exibidas como uma lista horizontal quando exibidas no modo de visualização ou ao executar um caso. Para grupos de verificação que possuem um grande número de opções, isso pode ocupar muito espaço e forçar o usuário a percorrer um formulário longo. Para mudar de uma lista vertical de opções para uma lista horizontal, adicione a classe "radio-inline" às DIVs ao redor de cada caixa de seleção.

$ ( "#selectCountries" ) . find ( "div.checkbox" ) . addClass ( "radio-inline" ) ;

Remova a caixa de sombra ao redor das caixas de seleção

Para remover a caixa de sombra azul clara ao redor das caixas de seleção e grupos de seleção, adicione o seguinte JavaScript ao formulário dinâmico:
